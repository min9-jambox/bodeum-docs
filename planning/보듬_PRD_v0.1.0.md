## 1. 제품 개요

### 1.1 한 줄 정의

요양보호사가 음성 1~2분으로 말하면, AI가 기관 평가 대비용 상태변화기록지 초안과 보호자 카카오 알림톡을 동시에 자동 생성하는 B2B SaaS.

### 1.2 제품 비전과 단계별 목표

보듬은 3단계에 걸쳐 **재가 장기요양 현장의 디지털 기록 인프라**로 성장한다.

| 단계 | 기간 | 핵심 목표 | 성공 기준 |
|:---|:---|:---|:---|
| **1단계 — 핵심** | 2026.04 ~ 2026.10 | 음성 → AI 초안 → 알림톡 파이프라인을 현장에서 완전히 동작하는 수준까지 완성 | 파이프라인 성공률 95%+, 시범 운영 5개소, AI 채택률 80%+ |
| **2단계 — 성장** | 2026.11 ~ 2027.06 | 유료 전환 + 데이터 기반 부가 가치 제공 + 다국어 고도화 | 유료 100개소+, MRR 500만 원+, 외국인 억양 인식률 90%+ |
| **3단계 — 확장** | 2027.07 ~ | 공단 엑셀 연동 + 요양 데이터 플랫폼화 + 전국 확장 | 유료 1,000개소+, 공단 엑셀 연동·ERP 호환 완료 |

### 1.4 경쟁사 분석 기반 차별화 전략

> 패밀리케어(패밀리노트) ERP 체험판 111장 화면 전수 분석 결과 도출. 상세 분석 → `패밀리케어_ERP_화면분석.md`

패밀리케어는 재가 장기요양기관 운영의 전 업무를 커버하는 올인원 ERP다. 9개 대메뉴, 40개+ 세부 화면으로 수급자 관리부터 급여·세무까지 지원한다. 그러나 **서술형 기록(상태변화기록지, 상담일지, 사례회의록)은 전부 수기 타이핑**이며, AI 자동화나 음성 입력은 전혀 없다.

**보듬이 이기는 포인트:**

| 영역 | 패밀리케어 | 보듬 차별점 |
|:---|:---|:---|
| 상태변화 기록지 | 수기 타이핑 | AI 음성 → 자동 작성 |
| 보호자 소통 | 행정 알림톡 (일정·비용만) | 케어 내용 AI 요약 알림 (감정적 가치) |
| 상담일지 | 수기 작성 | 음성 기록 연계 자동 생성 |
| 사례회의록 | 수기 작성 | 회의 녹음 → 자동 회의록 |
| 데이터 분석 | 단순 그래프 | AI 인사이트 (건강 추이, 리스크 알림) |
| UX | 데스크톱 웹 (테이블 중심) | 모바일 퍼스트 (음성 중심) |

**보듬이 배워야 할 포인트 (장기 로드맵 반영):**

| 패밀리케어 강점 | 보듬 반영 시점 |
|:---|:---|
| 공단 엑셀 기반 일정계획·청구내역 양방향 연동 (API 미제공, 엑셀 다운로드/업로드 방식) | 3단계 |
| 거의 모든 화면에 PDF/인쇄 출력 기능 | 1단계부터 (기관 평가 대비 PDF 출력 필수) |
| 일일/주간/월간 업무 체크리스트 가이드 | 3단계 (AI 자동 추천으로 차별화) |
| 일정·급여명세서·영수증 알림톡 | 2단계부터 알림톡 용도 확장 |
| 시급/월급 자동 계산, 4대보험 관리 | 3단계 올인원 전환 시 |

## 2. 사용자 정의 및 역할

### 2.1 사용자 유형

| 사용자 | 역할 | 접점 | 핵심 니즈 |
|:---|:---|:---|:---|
| **요양보호사** | 음성 녹음 → AI 초안 확인 → 제출 | 모바일 앱 (iOS/Android) | 타이핑 0, 1~2분 안에 끝나야 함 |
| **사회복지사 / 원장님** | AI 초안 검토 → 수정 → 확정 / 센터·수급자 관리 | 웹 백오피스 | 기관 평가 대비 기록 품질 보장 |
| **보호자** | 돌봄 알림 수신 (수동적) | 카카오 알림톡 | 앱 설치 없이 부모님 안부 확인 |
| **시스템 관리자** | 센터 온보딩, 전체 모니터링 | 어드민 콘솔 (웹) | 장애 조기 감지, 센터 관리 |

### 2.2 사용자 특성 — 설계 제약

요양보호사의 66.1%가 60세 이상이다. 이 사실이 모든 UX 의사결정의 출발점이다.

- **화면 요소는 최소화**: 한 화면에 하나의 행동만 요구
- **폰트 크기 최소 18sp**: 시력이 좋지 않은 고령 사용자 고려
- **색상 대비 WCAG AA 이상**: 4.5:1 이상의 명암비
- **터치 타겟 최소 48dp**: 큰 손가락, 떨리는 손 고려
- **텍스트 입력 0**: 모든 입력은 음성 또는 버튼 탭으로 완결
- **한국어 UI**: 영어 UI 요소 완전 배제 (버튼 라벨, 에러 메시지 등)

## 3. 유저 시나리오

### 시나리오 1: 요양보호사 김순자 씨 (68세) — 방문 종료 후 음성 기록

**배경**: 김순자 씨는 수급자 박할머니 댁에서 2시간 방문요양을 마쳤다. 공단 앱에서 출퇴근 태그와 급여제공기록 입력은 이미 완료했다. 이제 상태변화기록지를 작성하고 보호자에게 안부를 전해야 한다.

**기존 방식 (보듬 이전)**:
1. 수첩에 메모한 내용을 보며 상태변화기록지를 수기 또는 타이핑으로 작성 (10~15분)
2. 개인 카카오톡으로 보호자에게 "오늘 어머님 잘 계세요~" 메시지를 직접 전송
3. 보호자가 추가 질문하면 퇴근 후에도 답변해야 하는 감정노동 발생

**보듬 사용 시**:

```
[앱 실행] → 홈 화면에 오늘 방문 예정 수급자 목록 표시
    ↓
[박할머니 카드 탭] → 음성 녹음 화면 전환
    ↓
[큰 빨간 원형 버튼 탭] → 녹음 시작
    ↓
김순자 씨 음성:
"오늘 박할머니 컨디션 좋으셨어요.
 식사도 잘 하셨고 반찬도 골고루 드셨어요.
 거동이 좀 불편하셔서 거실에서 안방까지 부축해 드렸는데
 어제보다 걸음걸이가 좋아지신 것 같아요.
 혈압은 130에 85였어요."
    ↓
[녹음 중지 버튼 탭] → "AI가 정리하고 있습니다" 로딩 (5~10초)
    ↓
[AI 초안 2개 동시 표시]
  ┌─ 상태변화기록지 초안 ─────────────────────────┐
  │ [신체] 식사 양호. 반찬 골고루 섭취.              │
  │        거실→안방 부축 이동. 전일 대비 보행 개선.  │
  │ [인지] 특이사항 없음.                            │
  │ [정서] 기분 양호. 밝은 표정.                      │
  │ [활력] 혈압 130/85mmHg.                          │
  └──────────────────────────────────────────────────┘
  ┌─ 보호자 알림톡 미리보기 ──────────────────────┐
  │ 🏠 오늘 어머님 돌봄 알림                        │
  │ 식사를 잘 하셨고 기분도 밝으셨습니다.            │
  │ 걸음걸이가 어제보다 나아지셨어요.                │
  │ 혈압은 130/85로 안정적입니다.                    │
  └──────────────────────────────────────────────────┘
    ↓
[김순자 씨: 내용 맞으니까 바로 '제출' 버튼 탭]
    ↓
"제출 완료!" → 홈 화면으로 복귀 (박할머니 카드에 ✅ 표시)
```

**소요 시간**: 음성 녹음 1분 30초 + AI 생성 대기 10초 + 확인·제출 5초 = **약 2분**

**만약 AI 초안이 부정확하다면? — 보정 흐름**:

```
[김순자 씨: AI가 "혈압 113/85"로 잘못 인식한 것을 발견]
    ↓
['내용 추가' 버튼 탭] → 보충 녹음 시작
    ↓
"아까 혈압이 113이 아니라 130이에요"
    ↓
[녹음 중지] → AI가 기존 초안 + 보충 내용을 합쳐 재생성 (5~10초)
    ↓
[수정된 초안 확인: "혈압 130/85mmHg"] → '제출' 버튼 탭
```

또는 타이핑에 익숙한 젊은 요양보호사라면 초안 텍스트를 직접 탭하여 "113"을 "130"으로 수정할 수도 있다.

**결과**:
- 상태변화기록지 초안 → 원장님 웹 백오피스에 즉시 전송
- 보호자 알림톡 → 카카오 알림톡으로 보호자 휴대폰에 자동 발송
- 김순자 씨는 추가 타이핑이나 보호자 카톡 응대 없이 다음 수급자 방문으로 이동

### 시나리오 2: 원장님 이미숙 씨 (52세) — 웹 백오피스에서 AI 초안 검토·확정

**배경**: 소규모 방문요양 센터(수급자 15명, 요양보호사 4명)를 운영하는 원장님. 사회복지사가 없어 기록 검토를 직접 담당한다. 오후 5시, 오늘 방문을 마친 요양보호사들의 AI 초안이 백오피스에 도착했다.

**보듬 사용 시**:

```
[웹 백오피스 로그인] → 대시보드: 오늘 제출 현황

  • 오늘 방문 건수: 12건

  • 확정 대기: 10건
    ↓
[확정 대기 목록 클릭] → 초안 리스트 (수급자명 · 요양보호사명 · 제출 시각)
    ↓
[박할머니 — 김순자 제출분 클릭] → 상세 화면
  ┌─────────────────────────────────────────────────┐
  │ 📋 상태변화기록지 초안                           │
  │ [신체] 식사 양호. 반찬 골고루 섭취. ...           │
  │                                                   │
  │ 📱 보호자 알림톡 (발송 완료 ✓)                   │
  │ "식사를 잘 하셨고 기분도 밝으셨습니다..."         │
  │                                                   │
  │ 🎧 원본 음성 [재생 ▶]                            │
  └─────────────────────────────────────────────────┘
    ↓
```

- 확정된 기록만 공식 보관 문서로 분류
- PDF/엑셀 다운로드로 기존 서류 관리 체계와 호환

### 시나리오 4: 원장님 이미숙 씨 — 센터 최초 온보딩

**배경**: 보듬 도입을 결정한 원장님. 첫 설정을 진행한다.

```
[보듬 웹사이트에서 센터 등록 신청 (원장님 전화번호 입력)]
    ↓
[어드민이 센터 승인 → 원장님 User + UserCenterRole(DIRECTOR) 자동 생성]
    ↓
[원장님에게 SMS 초대 발송 → 앱 설치 + 전화번호 OTP 기기 등록 + 생체/PIN 설정]
    ↓
[앱 인증 완료 후 웹 백오피스 접속 → 앱 푸시 승인으로 웹 로그인]
    ↓
온보딩 위자드:
  Step 1. 센터 기본 정보 입력 (센터명, 주소, 대표 전화번호)
  Step 2. 카카오 알림톡 발신 프로필 설정 (카카오 비즈니스 채널 연동)
  Step 3. 수급자 등록 (이름, 생년월일, 보호자 연락처)
  Step 4. 요양보호사 등록 (이름, 연락처 → 앱 초대 링크 발송)
         * 이미 보듬에 가입된 요양보호사면 해당 센터에 소속만 추가
  Step 5. 수급자-요양보호사 배정
    ↓
[요양보호사에게 앱 초대 SMS 발송]
    ↓
요양보호사: 링크 탭 → 앱스토어/플레이스토어 → 설치 → 전화번호 OTP 인증 → 완료
(이미 앱이 설치된 요양보호사는 자동으로 새 센터가 목록에 추가됨)
```

**이미숙 원장님이 두 번째 센터를 추가하는 경우**:
```
[보듬 웹에서 추가 센터 등록 신청 (동일한 전화번호)]
    ↓
[어드민 승인 → 기존 User에 새 UserCenterRole(DIRECTOR) 추가]
    ↓
[백오피스 로그인 시 센터 선택 화면 표시]
    ↓
[센터 A / 센터 B 중 선택 → 해당 센터의 대시보드 진입]
[상단 센터 전환 드롭다운으로 실시간 전환 가능]
```

### 시나리오 5: 요양보호사 응우옌 티 란 씨 (32세, 베트남 출신) — 한국어 억양 음성 기록

**배경**: E-7 비자로 요양보호사 자격을 취득한 베트남 출신 종사자. 한국어로 의사소통은 가능하지만 타이핑은 매우 어렵다.

**보듬 사용 시**:

```
[앱에서 수급자 선택 → 녹음 버튼 탭]
    ↓
란 씨 음성 (베트남 억양 한국어):
"오늘 할머니 밥 잘 먹었어요.
 오늘 기분 좋아요, 많이 웃었어요.
 혈압 측정했어요, 125에 80이에요."
    ↓
[Whisper가 한국어로 자동 인식·변환]
    ↓
[AI 초안 생성 → 확인 → 제출]
```

**핵심 포인트**:
- Whisper는 다양한 억양의 한국어를 자동 인식 (초기에는 기본 동작 수준, 시범 운영에서 정확도 검증)
- 타이핑이 불가능한 외국인 요양보호사에게 음성 입력은 유일한 디지털 기록 수단
- 초기 수집된 외국인 억양 데이터를 기반으로 이후 Fine-tuning을 통한 품질 고도화

#### F-AI-007: 상담일지 AI 자동 생성 연계 🆕

| 항목 | 상세 |
|:---|:---|
| 설명 | AI가 음성 기록을 분석할 때, 상담 관련 내용(보호자 요청, 서비스 변경 논의, 수급자 불만 등)이 감지되면 상담일지 항목도 함께 자동 생성한다. 상태변화기록지와 동일한 LLM 호출에서 JSON 구조로 동시 반환하여 추가 비용 없이 구현한다. |

| 수락 기준 | STT 원문과 AI 초안 간 의미 정합성 점수(0~100) 산출. 점수 70 미만 시 "AI가 내용을 정확히 이해하지 못했을 수 있습니다" 경고 표시. 환각률 5% → 2% 미만으로 개선. |

#### F-ERP-001: 공단 엑셀 연동 및 ERP 호환

| 항목 | 상세 |
|:---|:---|
| 설명 | 국민건강보험공단 장기요양정보시스템과 엑셀 파일 기반 데이터 연동. 공단은 운영 데이터(일정계획, 청구내역, 급여제공기록) API를 제공하지 않으며, 공단 사이트에서 엑셀 다운로드 → 보듬에 업로드, 보듬에서 엑셀 내보내기 → 공단에 업로드하는 양방향 엑셀 교환 방식으로 연동한다. RFID 급여제공기록은 공단 앱(스마트 장기요양)에서 NFC로 처리하며 외부 API가 없으므로, 공단에서 다운로드한 RFID 실적 엑셀을 보듬에서 읽어와 매칭하는 방식으로 구현한다. |
| 우선순위 | P1 (3단계) |
| 수락 기준 | 공단 엑셀 양식(일정계획, 청구내역상세) 파싱·임포트 기능. 보듬 데이터를 공단 엑셀 양식으로 자동 변환·내보내기. RFID 실적 엑셀 임포트 및 수급자별 서비스 매칭. 주요 ERP(패밀리케어, 롱텀케어, 케어포 등) 엑셀 양식 호환 임포트/익스포트. |
| 경쟁사 대비 | 패밀리케어, 롱텀케어, 케어포 등 모든 주요 ERP가 공단 엑셀 다운로드/업로드 방식으로 연동 중. 이지케어만 웹 스크래핑 기반 "실시간 연동"을 표방하나 공식 API는 아님. 보듬은 엑셀 연동을 기본으로 하되, 장기적으로 자동화(파일 감시·일괄 처리) 수준을 높여 차별화. |
