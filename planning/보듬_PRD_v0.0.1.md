## 1. 제품 개요

### 1.1 한 줄 정의

요양보호사가 음성 1~2분으로 말하면, AI가 기관 평가 대비용 상태변화기록지 초안과 보호자 카카오 알림톡을 동시에 자동 생성하는 B2B SaaS.

### 1.2 제품 비전과 단계별 목표

보듬은 3단계에 걸쳐 **재가 장기요양 현장의 디지털 기록 인프라**로 성장한다.

| 단계 | 기간 | 핵심 목표 | 성공 기준 |
|:---|:---|:---|:---|
| **1단계 — 핵심** | 2026.04 ~ 2026.10 | 음성 → AI 초안 → 알림톡 파이프라인을 현장에서 완전히 동작하는 수준까지 완성 | 파이프라인 성공률 95%+, 시범 운영 5개소, AI 채택률 80%+ |
| **2단계 — 성장** | 2026.11 ~ 2027.06 | 유료 전환 + 데이터 기반 부가 가치 제공 + 다국어 고도화 | 유료 100개소+, MRR 500만 원+, 외국인 억양 인식률 90%+ |
| **3단계 — 확장** | 2027.07 ~ | 공단 연동 + 요양 데이터 플랫폼화 + 전국 확장 | 유료 1,000개소+, 공단 앱/ERP 연동 완료 |

## 2. 사용자 정의 및 역할

### 2.1 사용자 유형

| 사용자 | 역할 | 접점 | 핵심 니즈 |
|:---|:---|:---|:---|
| **요양보호사** | 음성 녹음 → AI 초안 확인 → 제출 | 모바일 앱 (iOS/Android) | 타이핑 0, 1~2분 안에 끝나야 함 |
| **사회복지사 / 원장님** | AI 초안 검토 → 수정 → 확정 / 센터·수급자 관리 | 웹 백오피스 | 기관 평가 대비 기록 품질 보장 |
| **보호자** | 돌봄 알림 수신 (수동적) | 카카오 알림톡 | 앱 설치 없이 부모님 안부 확인 |
| **시스템 관리자** | 센터 온보딩, 전체 모니터링 | 어드민 콘솔 (웹) | 장애 조기 감지, 센터 관리 |

### 2.2 사용자 특성 — 설계 제약

요양보호사의 66.1%가 60세 이상이다. 이 사실이 모든 UX 의사결정의 출발점이다.

- **화면 요소는 최소화**: 한 화면에 하나의 행동만 요구
- **폰트 크기 최소 18sp**: 시력이 좋지 않은 고령 사용자 고려
- **색상 대비 WCAG AA 이상**: 4.5:1 이상의 명암비
- **터치 타겟 최소 48dp**: 큰 손가락, 떨리는 손 고려
- **텍스트 입력 0**: 모든 입력은 음성 또는 버튼 탭으로 완결
- **한국어 UI**: 영어 UI 요소 완전 배제 (버튼 라벨, 에러 메시지 등)

## 3. 유저 시나리오

### 시나리오 1: 요양보호사 김순자 씨 (68세) — 방문 종료 후 음성 기록

**배경**: 김순자 씨는 수급자 박할머니 댁에서 2시간 방문요양을 마쳤다. 공단 앱에서 출퇴근 태그와 급여제공기록 입력은 이미 완료했다. 이제 상태변화기록지를 작성하고 보호자에게 안부를 전해야 한다.

**기존 방식 (보듬 이전)**:
1. 수첩에 메모한 내용을 보며 상태변화기록지를 수기 또는 타이핑으로 작성 (10~15분)
2. 개인 카카오톡으로 보호자에게 "오늘 어머님 잘 계세요~" 메시지를 직접 전송
3. 보호자가 추가 질문하면 퇴근 후에도 답변해야 하는 감정노동 발생

**보듬 사용 시**:

```
[앱 실행] → 홈 화면에 오늘 방문 예정 수급자 목록 표시
    ↓
[박할머니 카드 탭] → 음성 녹음 화면 전환
    ↓
[큰 빨간 원형 버튼 탭] → 녹음 시작
    ↓
김순자 씨 음성:
"오늘 박할머니 컨디션 좋으셨어요.
 식사도 잘 하셨고 반찬도 골고루 드셨어요.
 거동이 좀 불편하셔서 거실에서 안방까지 부축해 드렸는데
 어제보다 걸음걸이가 좋아지신 것 같아요.
 혈압은 130에 85였어요."
    ↓
[녹음 중지 버튼 탭] → "AI가 정리하고 있습니다" 로딩 (5~10초)
    ↓
[AI 초안 2개 동시 표시]
  ┌─ 상태변화기록지 초안 ─────────────────────────┐
  │ [신체] 식사 양호. 반찬 골고루 섭취.              │
  │        거실→안방 부축 이동. 전일 대비 보행 개선.  │
  │ [인지] 특이사항 없음.                            │
  │ [정서] 기분 양호. 밝은 표정.                      │
  │ [활력] 혈압 130/85mmHg.                          │
  └──────────────────────────────────────────────────┘
  ┌─ 보호자 알림톡 미리보기 ──────────────────────┐
  │ 🏠 오늘 어머님 돌봄 알림                        │
  │ 식사를 잘 하셨고 기분도 밝으셨습니다.            │
  │ 걸음걸이가 어제보다 나아지셨어요.                │
  │ 혈압은 130/85로 안정적입니다.                    │
  └──────────────────────────────────────────────────┘
    ↓
[김순자 씨: 내용 맞으니까 바로 '제출' 버튼 탭]
    ↓
"제출 완료!" → 홈 화면으로 복귀 (박할머니 카드에 ✅ 표시)
```

**소요 시간**: 음성 녹음 1분 30초 + AI 생성 대기 10초 + 확인·제출 5초 = **약 2분**

**만약 AI 초안이 부정확하다면? — 보정 흐름**:

```
[김순자 씨: AI가 "혈압 113/85"로 잘못 인식한 것을 발견]
    ↓
['내용 추가' 버튼 탭] → 보충 녹음 시작
    ↓
"아까 혈압이 113이 아니라 130이에요"
    ↓
[녹음 중지] → AI가 기존 초안 + 보충 내용을 합쳐 재생성 (5~10초)
    ↓
[수정된 초안 확인: "혈압 130/85mmHg"] → '제출' 버튼 탭
```

또는 타이핑에 익숙한 젊은 요양보호사라면 초안 텍스트를 직접 탭하여 "113"을 "130"으로 수정할 수도 있다.

**결과**:
- 상태변화기록지 초안 → 원장님 웹 백오피스에 즉시 전송
- 보호자 알림톡 → 카카오 알림톡으로 보호자 휴대폰에 자동 발송
- 김순자 씨는 추가 타이핑이나 보호자 카톡 응대 없이 다음 수급자 방문으로 이동

### 시나리오 2: 원장님 이미숙 씨 (52세) — 웹 백오피스에서 AI 초안 검토·확정

**배경**: 소규모 방문요양 센터(수급자 15명, 요양보호사 4명)를 운영하는 원장님. 사회복지사가 없어 기록 검토를 직접 담당한다. 오후 5시, 오늘 방문을 마친 요양보호사들의 AI 초안이 백오피스에 도착했다.

**보듬 사용 시**:

```
[웹 백오피스 로그인] → 대시보드: 오늘 제출 현황
    ↓
대시보드 요약:
  • 오늘 방문 건수: 12건
  • AI 초안 제출: 10건 (미제출 2건 — 김순자, 박영희 표시)
  • 확정 대기: 10건
    ↓
[확정 대기 목록 클릭] → 초안 리스트 (수급자명 · 요양보호사명 · 제출 시각)
    ↓
[박할머니 — 김순자 제출분 클릭] → 상세 화면
  ┌─────────────────────────────────────────────────┐
  │ 📋 상태변화기록지 초안                           │
  │ [신체] 식사 양호. 반찬 골고루 섭취. ...           │
  │                                                   │
  │ 📱 보호자 알림톡 (발송 완료 ✓)                   │
  │ "식사를 잘 하셨고 기분도 밝으셨습니다..."         │
  │                                                   │
  │ 🎧 원본 음성 [재생 ▶]                            │
  └─────────────────────────────────────────────────┘
    ↓
[원장님: 내용 확인 후 '확정' 버튼 클릭]
(필요 시 텍스트 직접 수정 후 확정)
    ↓
확정된 기록 → 기관 내 보관 문서로 저장 (공단 평가 실사 대비)
```

**핵심 포인트**:
- 미제출 요양보호사에 대한 알림 (앱 푸시 또는 대시보드 표시)
- 원본 음성을 재생하여 AI 초안의 정확성을 교차 검증 가능
- 확정된 기록만 공식 보관 문서로 분류
- PDF/엑셀 다운로드로 기존 서류 관리 체계와 호환

### 시나리오 3: 보호자 최영수 씨 (45세) — 카카오 알림톡 수신

**배경**: 직장인. 80세 어머니(박할머니)가 방문요양을 받고 있다. 매번 요양보호사에게 카톡으로 물어보기가 미안했다.

**보듬 사용 시**:

```
[카카오톡 알림 수신] ← 별도 앱 설치 없음
    ↓
┌─ 카카오 알림톡 ────────────────────────────────┐
│ [보듬] 🏠 오늘 어머님 돌봄 알림                  │
│                                                   │
│ 📅 2026년 3월 18일 (수) 오후 3:00                │
│ 👤 담당: 김순자 요양보호사                        │
│                                                   │
│ 식사를 잘 하셨고 기분도 밝으셨습니다.             │
│ 걸음걸이가 어제보다 나아지셨어요.                 │
│ 혈압은 130/85로 안정적입니다.                     │
│                                                   │
│ 💛 OO방문요양센터                                 │
└───────────────────────────────────────────────────┘
```

**핵심 포인트**:
- 단방향 알림: 보호자가 직접 요양보호사에게 채팅할 수 없음 → 감정노동 차단
- 앱 설치 0: 카카오톡만 있으면 수신
- 알림톡은 센터의 카카오 채널(채널추가형)로 발송 → 오픈율 90% 기준

### 시나리오 4: 원장님 이미숙 씨 — 센터 최초 온보딩

**배경**: 보듬 도입을 결정한 원장님. 첫 설정을 진행한다.

```
[보듬 웹사이트에서 센터 등록 신청 (원장님 전화번호 입력)]
    ↓
[어드민이 센터 승인 → 원장님 User + UserCenterRole(DIRECTOR) 자동 생성]
    ↓
[원장님에게 SMS 초대 발송 → 앱 설치 + 전화번호 OTP 기기 등록 + 생체/PIN 설정]
    ↓
[앱 인증 완료 후 웹 백오피스 접속 → 앱 푸시 승인으로 웹 로그인]
    ↓
온보딩 위자드:
  Step 1. 센터 기본 정보 입력 (센터명, 주소, 대표 전화번호)
  Step 2. 카카오 알림톡 발신 프로필 설정 (카카오 비즈니스 채널 연동)
  Step 3. 수급자 등록 (이름, 생년월일, 보호자 연락처)
  Step 4. 요양보호사 등록 (이름, 연락처 → 앱 초대 링크 발송)
         * 이미 보듬에 가입된 요양보호사면 해당 센터에 소속만 추가
  Step 5. 수급자-요양보호사 배정
    ↓
[요양보호사에게 앱 초대 SMS 발송]
    ↓
요양보호사: 링크 탭 → 앱스토어/플레이스토어 → 설치 → 전화번호 OTP 인증 → 완료
(이미 앱이 설치된 요양보호사는 자동으로 새 센터가 목록에 추가됨)
```

**이미숙 원장님이 두 번째 센터를 추가하는 경우**:
```
[보듬 웹에서 추가 센터 등록 신청 (동일한 전화번호)]
    ↓
[어드민 승인 → 기존 User에 새 UserCenterRole(DIRECTOR) 추가]
    ↓
[백오피스 로그인 시 센터 선택 화면 표시]
    ↓
[센터 A / 센터 B 중 선택 → 해당 센터의 대시보드 진입]
[상단 센터 전환 드롭다운으로 실시간 전환 가능]
```

### 시나리오 5: 요양보호사 응우옌 티 란 씨 (32세, 베트남 출신) — 한국어 억양 음성 기록

**배경**: E-7 비자로 요양보호사 자격을 취득한 베트남 출신 종사자. 한국어로 의사소통은 가능하지만 타이핑은 매우 어렵다.

**보듬 사용 시**:

```
[앱에서 수급자 선택 → 녹음 버튼 탭]
    ↓
란 씨 음성 (베트남 억양 한국어):
"오늘 할머니 밥 잘 먹었어요.
 오늘 기분 좋아요, 많이 웃었어요.
 혈압 측정했어요, 125에 80이에요."
    ↓
[Whisper가 한국어로 자동 인식·변환]
    ↓
[AI 초안 생성 → 확인 → 제출]
```

**핵심 포인트**:
- Whisper는 다양한 억양의 한국어를 자동 인식 (초기에는 기본 동작 수준, 시범 운영에서 정확도 검증)
- 타이핑이 불가능한 외국인 요양보호사에게 음성 입력은 유일한 디지털 기록 수단
- 초기 수집된 외국인 억양 데이터를 기반으로 이후 Fine-tuning을 통한 품질 고도화

### 6.2 보안

#### 계층형 인증 정책 (Tiered Authentication)

| 역할 | 최초 인증 | 일상 인증 | 세션 유지 | 재인증 트리거 |
|:---|:---|:---|:---|:---|
| **요양보호사 (앱)** | SMS OTP → 기기 등록 → 생체/PIN 설정 | 생체/PIN (0.5초) | 90일 (기기 신뢰 기반) | 새 기기, 90일 미사용, 비정상 접근 |
| **원장님 (웹)** | 보듬 앱 필수 설치 → 앱 인증 완료 상태 | 앱 푸시 승인 (생체/PIN) | 14일 (브라우저 신뢰) | 14일 만료, 새 브라우저, 비정상 접근 |
| **어드민 (웹)** | 사전 등록 전화번호 + 앱 인증 완료 | 앱 푸시 승인 + SMS OTP (2FA) | 14일 (브라우저 신뢰) | 매 로그인 시 2FA 필수 |

> **설계 원칙 — "보듬 앱이 곧 열쇠"**: 모바일 앱이 전체 시스템의 유일한 인증기(Authenticator)로 동작한다. 웹 로그인은 앱 푸시 승인을 기본 경로로, SMS OTP는 폴백으로만 사용한다. 이를 통해 별도 인증 수단(카카오, 이메일 등) 없이 단일 인프라로 전체 인증을 처리한다.

#### 기타 보안 요구사항

| 항목 | 요구사항 |
|:---|:---|
| 통신 | 모든 API 호출 HTTPS (TLS 1.3) |
| 토큰 | JWT (Access Token 15분 + Refresh Token: 앱 90일 / 웹 14일). JWT 페이로드에 user_id, active_center_id, role, device_trusted, mfa_verified(어드민) 포함. |
| 기기 신뢰 | 기기 등록 시 device_id + device_fingerprint를 서버에 저장. 등록된 기기에서만 생체/PIN 인증 허용. 기기 분실 시 원격 해제 가능. |

### 6.5 SMS 비용 최적화 근거

계층형 인증 설계의 핵심 동기 중 하나는 SMS OTP 비용의 구조적 절감이다.

| 시나리오 | SMS 발송 빈도 | 월 비용 (100개소 기준, 건당 16원) |
|:---|:---|:---|
| **기존 방식** (매일 SMS OTP 로그인) | 요양보호사 1,000명 × 22일 = 22,000건/월 | ~35만 원/월 |
| **계층형 인증 (현재 설계)** | 최초 기기 등록 1회 + 기기 교체·재인증 소량 | ~1만 원 이하/월 |

> 비용 절감 원리: 일상 로그인은 기기에 저장된 생체/PIN으로 처리하고, 웹 로그인은 FCM 푸시(무료)로 처리하므로 SMS는 최초 등록 및 폴백 상황에서만 사용된다. 사업 성장에 따라 센터 수가 증가해도 SMS 비용은 선형 증가하지 않는 구조이다.

```
- 신체 상태: 식사, 이동, 배설, 활력징후 등
- 인지 상태: 의사소통, 기억력, 판단력 등
- 정서 상태: 기분, 감정, 행동 변화 등
- 특이사항: 위 3개 영역에 해당하지 않는 관찰 사항

[수급자 컨텍스트]
{수급자 기본 정보: 연령, 장기요양등급, 주요 질환}

[핵심 규칙]
1. 음성에서 언급되지 않은 내용은 절대 추가하지 마시오. (환각 방지)
2. 의료적 판단이나 진단을 내리지 마시오. 관찰 사실만 기록.
3. 각 영역에 해당 내용이 없으면 "특이사항 없음"으로 명시.
4. 존댓말 사용. 간결한 서술체.
5. 혈압, 체온 등 수치는 그대로 기재.

[출력 형식]
JSON으로 출력:
{
  "care_record": {
    "physical": "...",
    "cognitive": "...",
    "emotional": "...",
    "special_notes": "..."
  },
  "alimtalk_message": "..."
}

[알림톡 메시지 규칙]
1. 200자 이내.
2. 의료 전문 용어 사용 금지.
3. 따뜻하고 안심되는 톤.
4. 수치(혈압 등)는 포함하되 해석하지 않음.
5. "~하셨습니다", "~하셨어요" 체.
```

### 7.2 프롬프트 품질 관리

- 정식 운영 전 50건 이상의 모의 음성으로 프롬프트 튜닝
- 환각률 측정: 원본 음성에 없는 내용이 생성된 비율 → 1단계 목표 5% 미만 → 2단계 목표 2% 미만
- 채택률 측정: 원장님이 수정 없이 확정한 비율 → 목표 80%+
- 운영 기간 동안 주 1회 프롬프트 버전 업데이트. 운영 안정화 후 월 1회 정기 튜닝
- 2단계에서 음성-텍스트 정합성 검증 엔진(F-AI-005)과 연동하여 자동 품질 모니터링 체계 구축
