# 📋 사업기획서 2.0: 보듬 (Bodeum)
> **"행정은 AI에게, 돌봄은 사람에게, 안심은 가족에게."**
> 고령 요양보호사·외국인 종사자의 기록 노동과 언어 장벽을 'AI Zero-Typing'으로 해결하는 시니어 테크 B2B SaaS 플랫폼

---

## 1. Executive Summary (사업 요약)

초고령화 시대, 전국 현직 요양보호사의 60세 이상 비율은 66.1%입니다(국민건강보험공단 요양보호사 실태조사, 2024년 7월 기준). 이들은 필수인 건보공단 앱 외에도 보호자 응대를 위한 **개인 카카오톡 장문 전송(감정 노동)**과 기관 평가를 위한 **주 1회 서술형 관찰일지 수기 작성**이라는 삼중 고통에 시달립니다.

**보듬**은 이 문제를 두 개의 핵심 기술로 해결합니다.

첫째, **AI Voice Zero-Typing**: 퇴근 전 사투리나 외국어로 1~2분 음성 녹음 한 번으로 ① 평가 제출용 상태변화기록지(행정 서식)와 ② 보호자용 감성 알림톡을 동시에 자동 생성합니다. 음성 외 현장 사진·짧은 동영상도 함께 첨부할 수 있습니다.

둘째, **GPS 지오펜싱 넛지 UX**: 요양보호사가 수급자 댁 50m 이내 진입·이탈 시 앱이 먼저 화면을 깨워 기록을 유도합니다. 요양보호사는 더 이상 기록을 잊지 않습니다.

거대 ERP들과 정면충돌하지 않습니다. 기존 ERP 위에 얹히는 **가벼운 서브 플러그인(Add-on)** 포지셔닝으로, 원장님은 기존 시스템을 그대로 쓰면서 보듬이 생성한 결과물을 엑셀로 받아 기존 ERP에 붙여 넣기만 하면 됩니다.

**핵심 수익 모델:** 센터당 월 10,000원 구독 (20인 이하 기준). 전국 재가장기요양기관 약 22,700개소 중 약 4%(약 800곳) 확보 시 월 순수익 500만 원 자동화 달성.

## 2. 문제 정의 (Problem)

### 2-1. 삼중 행정 고통 구조

| 채널 | 목적 | 고통 |
|:---|:---|:---|
| 건보공단 앱 (W4C) | 급여 청구 필수 태그·서명 | 법적 필수, 대체 불가 |
| 개인 카카오톡 | 보호자 안심 소통 | 사생활 침해, 24시간 감정 노동 |
| 수기/ERP 타이핑 | 주 1회 서술형 관찰일지 | 60대에게 타자 입력은 극한 스트레스 |

공단 앱은 O/X 체크 방식이므로 **서술형 평가 제출용 기록지는 별도로 반드시 작성**해야 합니다. 이 이중 작업(Double Entry)이 요양보호사 이직의 핵심 원인 중 하나입니다.

### 2-2. 디지털 소외: 고령 종사자 + 외국인 종사자

**[고령자 문제 — 팩트 기반]**
- 2024년 7월 기준 전체 현직 요양보호사 66만 명 중 60세 이상 비율 **66.1%**, 70대 이상만 10만 명에 달함 (국민건강보험공단 NHIS, 2024 요양보호사 실태조사)
- 요양보호사 자격증 보유자는 약 291만 명이지만, 실제 종사자는 22.8%인 약 66만 명에 불과 (KOSIS 국가통계포털, 2023~2024 장기요양 인력통계)

**[외국인 종사자 문제]**
- 법무부가 2024년 7월 E-7 특정활동 비자에 '요양보호사' 직종을 신설. 국내 대학 졸업 외국인 유학생 대상으로 연 400명 쿼터 내 시범 운영 중. (법무부 비자 직종 신설 공고, 2024년 7월)
- 서술형 한국어 기록 작성이 구조적으로 어려움 → 부실 기록으로 인한 수가 환수 리스크 발생

### 2-3. 경쟁사들이 해결하지 못한 맹점
케어로그, 패밀리노트 등 기존 소통 앱은 모두 **"정보를 누가 입력하는가"** 문제를 해결하지 않습니다. 앱이 예쁠수록 60대 요양보호사에게는 채팅 민원창이 하나 더 늘어날 뿐입니다. 보듬은 **입력 행위 자체를 소멸**시킵니다.

## 3. 솔루션 및 핵심 기능 (Solution)

### 3-1. 핵심 기술 스택

- **음성 인식**: OpenAI Whisper (사투리·다국어 지원)
- **텍스트 생성**: GPT-4o-mini (경량 모델 활용, AI 처리 단가 최소화)
- **보호자 알림**: 카카오 알림톡 API (건당 약 9~15원, 솔라피·NHN Cloud 등 발송사 기준)
- **위치 감지**: GPS 지오펜싱 (절전 모드, 반경 50m)

### 3-2. 코어 플로우

```
[요양보호사] 수급자 댁 50m 이내 진입
    → 앱 자동 알림: "어르신 댁 도착! 녹음 시작 🎤"
    → 사투리·외국어로 음성 녹음 (30초~1분) + 사진·동영상 선택 첨부
    → AI 자동 변환:
        ① 평가용 상태변화기록지 (행정 서식, 엑셀 저장)
        ② 보호자용 감성 카톡 (카카오 알림톡 발송)
[원장님] 월말 원클릭 엑셀 다운로드 → 기존 ERP에 업로드
```

### 3-3. 주요 기능

**[요양보호사용]**
- **Zero-Typing 음성 기록**: 타자 없이 사투리·외국어 음성 녹음(30초~1분)으로 행정 서류 자동 완성. 사진·동영상 첨부 지원.
- **GPS 지오펜싱 넛지**: 출근·퇴근 시점에 앱이 먼저 화면을 깨워 기록 유도. GPS 꺼진 경우 수동 버튼으로 백업.
- **다국어 기록 엔진**: 모국어로 말해도 AI가 한국어 표준 서식으로 번역·정형화.

**[원장님용]**
- **마법의 엑셀 다운로드**: 기존 ERP 호환 양식으로 원클릭 다운로드. 이중 작업 논란 기술적 종식.
- **스케줄 엑셀 업로드 (1회/월)**: 기존 ERP에서 뽑은 월간 스케줄 엑셀을 드래그 앤 드롭으로 업로드.
- **프리미엄 마케팅 리포트**: 원장님 직인 + 어르신 활동 사진 + AI 감성 요약 → 보호자 카톡 전송.

**[보호자용]**
- **무설치 알림톡**: 별도 앱 설치·회원가입 없이 카카오톡으로 매일 리포트 수신.
- **GPS 방문 인증 뱃지**: "실제 선생님이 다녀갔음"을 증명하는 인증 마크.

### 3-4. 멀티모달 데이터 정합성 검증 (기술 차별화 및 특허 핵심)
음성과 사진 데이터 간 논리적 모순을 AI가 자동 감지합니다. (예: 음성 "식사 다 하셨다" + 사진 "잔반 가득") 신뢰도 임계값 미달 시 요양보호사에게 재확인 UI 제공.

## 5. 비즈니스 모델 (Business Model)

### 5-1. 과금 구조

- 수급자 20인 이하: 월 **10,000원**
- 초과 10명당: 월 **10,000원** 추가
- 알림톡 발송 비용: **실비 종량제 (센터 직접 충전)**
- 프리미엄 업셀링: 원장님 직인 + 사진 갤러리 + GPS 방문 인증 → 원장님에게 업셀링 (보호자 무과금 원칙)

### 5-2. 초기 프로모션 구조

| 단계 | 내용 |
|:---|:---|
| **도입 첫 달** | 기본료 + 알림톡 발송비 **전액 무료** (예창패 마케팅 예산 활용) |
| **프로모션 기간 (6개월)** | 첫 달 무료 + 이후 6개월 월 **5,000원** |
| **정상화 이후** | 첫 달 무료 + 이후 월 **10,000원** |

### 5-3. 수익성 시뮬레이션 (BEP)

> ※ 아래 원가는 실제 API 단가 및 서버 트래픽 추정치 기반이며, 실서비스 운영 후 재검증 필요

| 항목 | 값 | 근거 |
|:---|:---|:---|
| 센터 1곳 월 매출 | 10,000원 | 기본 구독료 |
| 알림톡 발송비 (200건/월 기준) | ~1,800원 | 건당 약 9원 × 200건 (NHN Cloud 기준) |
| LLM API 비용 | ~200원 | GPT-4o-mini 경량 모델 기준 추정 |
| 서버 종량제 | ~500원 | AWS 등 클라우드 종량 추정 |
| **센터 1곳 공헌이익** | **~7,200원** | **마진율 약 72%** |
| 월 고정비 | ~300,000원 | 서버 기본료·도메인·예비비 |
| **월 순수익 500만 원 목표 센터 수** | **약 800개소** | 계산식: (5,300,000원 ÷ 7,200원) |
| 전국 재가기관 대비 필요 점유율 | **약 3.5~4.7%** | 22,735개소 또는 17,000개소 기준 |

## 6. 이해관계자별 핵심 프로세스 (User Journey)

### [원장님]
1. 기존 ERP에서 월간 스케줄 엑셀 추출 → 보듬 웹에 드래그 앤 드롭 (1회/월)
2. 요양보호사 앱에 스케줄 자동 연동 (GPS 넛지 자동 활성화)
3. 월말: 보듬에서 기존 ERP 호환 엑셀 원클릭 다운로드 → 기존 ERP 업로드

### [요양보호사]
1. 앱 최초 1회 설치 + 알림 권한 허용
2. 수급자 댁 50m 진입 시 앱 자동 알림 → 녹음 버튼 터치 → 사투리·외국어로 음성 녹음 + 사진·동영상 선택 첨부
3. AI가 행정용 텍스트 + 가족용 감성 카톡 자동 생성 → 확인 후 발송 버튼 1회

### [보호자]
1. 앱 설치 없음. 카카오톡으로 매일 알림 수신
2. 베이직: 텍스트 요약 리포트 / 프리미엄: 사진 갤러리 + GPS 인증 + 감성 요약

## 14. 기술 아키텍처 상세 (Technical Architecture)

### 14-1. 4단계 AI 파이프라인

```
[1단계: Edge Layer — 수집 및 비식별화]
  - 안면·실명 정보 즉시 마스킹 (ISMS 준수)
  - GPS + 타임스탬프 메타데이터 고정 (위변조 방지)
  - 모국어 음성 + 현장 사진 패키징

[2단계: Cognitive Layer — 다국어 컨텍스트 분석]
  Claude / GPT-4o-mini 기반:
  - 외국어 음성 맥락 파악 → 보건복지부 표준 용어로 자동 치환
  - 예: "할머니 죽을 천천히 드셨지만 다 비웠어요"
      → [식사: 일반식(죽), 섭취량: 100%, 상태: 저작 능력 저하 주의]

[3단계: Validation Layer — 멀티모달 정합성 검증 (핵심 특허 구간)]
  - 오차 20% 초과 시 즉시 사용자 알림
  - Integrity Score 미달 시 기록 확정 거부 → 수동 검토 전환

[4단계: Output Layer — 멀티 채널 결과 생성]
  ① 행정용: W4C 표준 규격 + 엑셀 다운로드
  ② 보호자용: AI 감성 요약 + 카카오 알림톡 발송
  ③ 글로벌용: CDM — 일본 Kaigo / 독일 DiPA 이식 가능
```

## 15. 외국인 요양보호사 전략 (Foreign Worker Strategy)

### 15-1. 현황 (팩트 기반)

현재 국내에서 활동하는 외국인 요양보호사는 전체 요양보호사 64만여 명 중 약 0.9% 수준인 **5,600명**에 불과하다. (보건복지부·국민건강보험연구원, 2025년 기준)

법무부는 2024년 7월 E-7 특정활동 비자에 '요양보호사' 직종을 신설하고, **연간 400명 쿼터** 내에서 시범 운영 중이다. 2025년 1월 21일 최초 3명(베트남·몽골·말리 국적)에게 비자를 발급했다. (법무부 공식 발표, 재외동포신문 2025.02.18 인용)

**⚠️ 팩트체크 경고**: "향후 5년 내 외국인 비중 10~20%" 전망은 현재 정책 속도(연 400명 쿼터, 실제 발급 3건)와 괴리됩니다. 외국인 수요는 중장기 성장 동력으로만 포지셔닝하고, 현재 핵심 타겟은 고령 내국인 요양보호사(66.1%)입니다.

### 15-2. 인력 부족 구조적 배경

2028년 전국에서 **11만 6,000여 명 부족** 전망. (건강보험연구원, 2023) 60대 이상이 66%, 20·30대는 1% 미만. 청년이 외면하는 '노인이 노인을 돌보는' 구조가 외국인 수요의 근거입니다.

### 15-3. 기술적 차별화

| 문제 | 보듬의 솔루션 |

| 한국어 기록 작성 불가 | 모국어로 말하면 AI가 한국어 표준 서식으로 자동 번역·정형화 |

## 18. 예비창업패키지 심사 대비 Q&A 전략

### Q1. "기존에도 유사한 요양 앱이 많은데, 왜 이 기술이 필요한가?"
> "케어로그·하이케어 모두 '입력창을 예쁘게 만들었을 뿐' 60대 요양보호사에게 타자 입력을 여전히 강요합니다. 저희는 입력 행위 자체를 소멸시킵니다. 사투리나 외국어로 1~2분 말하면 AI가 ① 공단 제출용 서술형 기록지와 ② 보호자 감동 카톡을 동시에 완성합니다."

### Q2. "AI 환각(Hallucination)이 발생하면 법적 증빙을 어떻게 하는가?"
> "결정론적 필터링(Deterministic Filtering)을 도입했습니다. '음성 → 사진 → W4C 법적 기준' 3단계 검증 루프를 거치며, 기준치 미달 시 기록이 확정되지 않고 수정을 강제합니다. 이는 30년 데이터베이스 설계 노하우를 AI 파이프라인에 이식한 구조입니다."

### Q3. "케어로그·하이케어 등 선점 경쟁사가 있는데 어떻게 이기는가?"
> "케어로그는 채팅 기능으로 요양보호사의 감정 노동을 앱 안으로 옮겼을 뿐입니다. 하이케어는 공단 행정 연동에 특화되어 있습니다. 저희는 이 두 가지가 하지 못한 '입력 소멸 + 단방향 감성 리포트 + 멀티모달 검증'의 조합으로 완전히 다른 레이어에서 경쟁합니다."

### Q4. "1인 개발자가 AI 기술을 감당할 수 있는가?"
> "자체 AI 모델 개발은 처음부터 배제합니다. GPT-4o-mini 등 상용 API를 극도로 가벼운 RAG 아키텍처로 조립합니다. 요양보호사 1인당 월 AI 처리 원가 500원 미만. 버티컬 요양 도메인 데이터 파이프라인을 선점하는 것이 진짜 해자(Moat)입니다."
