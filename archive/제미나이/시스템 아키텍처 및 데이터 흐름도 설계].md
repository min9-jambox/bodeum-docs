# [기술 상세] AI 멀티모달 데이터 파이프라인 아키텍처 (수정본)

## 1. 시스템 데이터 흐름 (Data Flow)
1. **수집 단계(Edge):** 음성(모국어), 사진(현장 객체), 영상 타임스탬프를 묶어 '데이터 패키지' 형성.
2. **분석 및 검증 단계(Cognitive & Validation):** AI 에이전트가 데이터 정합성을 검증(Conflict Resolution)하고 법적 기록 완성.
3. **데이터 표준화 저장 단계 (Export-Ready Store):** - 생성된 데이터를 보건복지부 요양급여 표준 서식(W4C) 규격에 맞게 변환하여 DB에 저장.
   - **(핵심)** 공공 API 부재 대응용 '표준 규격 익스포트 엔진' 탑재.
4. **전송 단계 (Multi-Channel Output):**
   - **기관용:** 표준 규격 데이터 시각화 리포트(PDF/엑셀).
   - **보호자용:** AI가 생성한 '감성 분석 기반 안심 알림톡' 발송.

## 2. 시스템 구성도 (아키텍처)
[Client Side: 제로-UI(음성/카메라)] 
      ↓ (데이터 패키지 전송)
[Middleware: 결정론적 필터 레이어 (환각 제어)] 
      ↓ (데이터 처리)
[Cognitive Engine: Claude 3.5 기반 다국어/엔티티 추출]
      ↓ (검증 통과 데이터 저장)
[Standardized DB: W4C 표준 규격 저장소] --+-- [기관용 리포트 도구]
                                       |
                                       +-- [알림톡 엔진: 보호자용 감성 메시지 발송]
