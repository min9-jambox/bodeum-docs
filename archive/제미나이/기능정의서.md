# [기능 상세 기획서] AI 기반 요양 기록 정합성 검증 플랫폼

## 1. 개요
* **목적:** 요양보호사의 기록 업무 자동화 및 멀티모달 데이터(음성, 이미지) 간 정합성 검증을 통한 기록의 신뢰성 확보.
* **적용 대상:** 국내 요양기관(재가/시설) 및 향후 글로벌 시장(일본, 독일 등).

## 2. 주요 기능 상세 명세

### A. 멀티모달 입력 및 처리 엔진 (Input & Processing)
1. **[Zero-UI 기반 수집]**
   - **기능:** 앱 실행 시 즉시 '음성 녹음' 및 '카메라 촬영' 모드 진입.
   - **기술 스펙:** 10초 이내 녹음/촬영 종료 시 서버 전송, 타임스탬프 기반 자동 매핑.
2. **[결정론적 필터링(Deterministic Filtering)]**
   - **기능:** 30년 개발 노하우 기반, AI가 생성한 기록이 보건복지부 표준 서식(W4C) 범위를 벗어나지 않도록 검증.
   - **로직:** 입력 데이터 → [필터링 규칙(도메인 사전)] → [LLM] → [정합성 검증] → 최종 결과값 도출.

### B. 정합성 검증 엔진 (Consistency Check Engine - 핵심)
1. **[데이터 비교 모듈]**
   - **기능:** 음성 텍스트 추출 값(식사량 100%)과 사진 분석값(잔반 감지) 간 논리적 모순 감지.
   - **알림:** "기록하신 식사량과 사진 속 잔반량이 일치하지 않습니다. 확인 후 수정해 주세요." 메시지 팝업.
2. **[표준 규격 정형화]**
   - **기능:** 비정형 대화문을 요양 표준 서식(식사, 투약, 배설 등)의 데이터 필드로 자동 매핑.
   - **저장:** 시스템 내부 'Export-Ready DB'에 규격화하여 저장(W4C 규격 준수).

### C. 보호자 안심 서비스 (Communication Engine)
1. **[AI 감성 리포트 생성]**
   - **기능:** 수집된 데이터를 바탕으로 보호자 맞춤형 언어(안심 메시지) 자동 생성.
   - **예시:** "오늘 할머니께서 평소보다 식사를 천천히 하셨지만, 끝까지 다 드셨습니다. 표정이 무척 밝으십니다."
2. **[알림톡 자동 전송]**
   - **기능:** 매일 정해진 시간에 요약본을 카카오톡 비즈니스 메시지로 전송.

### D. 글로벌 확장 기반 (Scalability)
1. **[공통 데이터 모델(CDM) 엔진]**
   - **기능:** 국가별 데이터 표준(일본 Kaigo, 독일 DiPA 등)을 추상화하여 저장 구조를 설계.
   - **확장성:** 2028년 글로벌 진출 시, 해당 국가의 규격 모듈만 추가하여 이식 가능하도록 설계.

## 3. 데이터 구조 및 표준 저장소 (Data Schema)
* **표준 규격 필드:** `Service_Type`, `Timestamp`, `Input_Data_Link`, `Standard_Entity`, `Integrity_Score`, `Validated_Status`.
* **운영 전략:** 현재 공공 API 부재 대응을 위해, W4C 표준 서식 규격의 JSON/CSV 익스포트 기능을 기본 지원.

## 4. 운영 정책 (Operational Policy)
* **데이터 관리:** 모든 데이터는 비식별화 처리 후 클라우드 전송.
* **에러 제어:** 정합성 점수(Integrity Score)가 일정 수준 미달 시 기록으로 확정하지 않고 수동 검토 프로세스 전환.
