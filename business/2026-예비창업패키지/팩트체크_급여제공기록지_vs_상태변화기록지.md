# 팩트체크: 급여제공기록지 vs 상태변화기록지 vs 희망이음

> 이 문서는 보듬 사업계획서의 핵심 논리("보듬이 자동화하는 상태변화기록지는 공단 시스템과 별개")를 뒷받침하는 팩트를 정리한 것이다.

---

## 핵심 결론

**"요양보호사 방문 후 원장님이 희망이음에 입력하는 것"은 사실과 다르다.**

원장님이 방문 후 관리하는 것은 **급여제공기록의 전송현황 확인**이며, 이는 **희망이음이 아니라 장기요양기관 업무지원시스템(공단 웹) 또는 스마트장기요양앱**에서 이루어진다. 그리고 이것은 **상태변화기록지와 완전히 다른 서류**이므로, 보듬의 사업 논리에 영향을 주지 않는다.

---

## 1. 세 가지 서류/시스템의 완전 분리 구조

### A. 급여제공기록지 (장기요양급여 제공기록지)

| 항목 | 내용 |
|:---|:---|
| **목적** | 수가 청구 (건보공단에 비용을 청구하기 위한 법적 근거 서류) |
| **법적 근거** | 「장기요양급여 제공기준 및 급여비용 산정방법 등에 관한 고시」(보건복지부고시 제2025-115호) 별지 서식 |
| **서식 수** | 14종 (2025년 리뉴얼 후. 방문요양·방문목욕·방문간호·주야간보호 등) |
| **기록 형태** | 체크리스트 + 시간 기록 위주 (서술형 아님) |
| **작성 주체** | 요양보호사 (방문 시 직접 작성) |
| **작성 시기** | 매 방문 당일 작성 원칙. 수급자/보호자 서명 필수 |
| **입력 채널** | ① 스마트장기요양앱 (모바일, RFID 전자태그 시작/종료 + 서비스 내용 전송) ② 장기요양기관 업무지원시스템 (공단 웹, 기관관리자 확인·관리) |
| **관리 주체** | 국민건강보험공단 |
| **제출 대상** | 공단 (월 단위 급여비용 청구 시 근거 자료) |
| **원장 역할** | 공단 웹에서 요양보호사의 RFID 전송현황 확인, 누락·오류건 수기 보완 관리 |

**출처**: 국가법령정보센터 「장기요양급여 제공기준 고시」 별표·서식; 케어포 재가급여전자관리시스템(RFID) 가이드북; 스마트장기요양앱 Google Play 설명

### B. 상태변화기록지 (수급자 상태변화기록지)

| 항목 | 내용 |
|:---|:---|
| **목적** | 기관 평가 대비 (3년 주기 공단 현장 실사) |
| **법적 근거** | 장기요양기관 평가 매뉴얼 평가지표 (법정 서식은 아니나, 평가 시 작성 충실도를 점수화) |
| **서식** | 자체 서식 (기관마다 다름). 서술형으로 작성 |
| **기록 형태** | 서술형 (수급자의 신체·인지·정서 상태를 구체적으로 서술) |
| **작성 주체** | 요양보호사가 작성/구술 → 사회복지사/원장이 취합·검토 |
| **작성 주기** | 주 1회 이상 (7일 간격. 하루라도 서비스 제공하면 작성) |
| **입력 채널** | 별도 전산 시스템 없음 (수기, 엑셀, 자체 양식 등 기관마다 상이) |
| **관리 주체** | 각 기관 자체 |
| **제출 대상** | 공단에 제출하지 않음. 기관 내 보관 후 평가 실사 시 현장에서 확인 |
| **원장 역할** | 매월 회수 후 수급자 파일에 보관. 기록 품질 검토 |

**출처**: 케어매니저 방문요양 업무 Q&A (care-m.co.kr); 두손누리 방문요양센터 상태변화기록지 작성 가이드; 2026년 장기요양기관 재가급여(방문요양) 평가 매뉴얼

**중요 보충**: 공단 검색 결과에 따르면, 급여제공기록지의 특이사항란을 아주 충실하게 작성할 경우 상태변화기록지 대신으로 인정받을 수 있는 경우도 있다. 그러나 이는 예외적이며, 대부분의 기관은 두 기록을 별도로 관리한다.

### C. 희망이음 (사회서비스정보시스템)

| 항목 | 내용 |
|:---|:---|
| **목적** | 사회복지시설 및 법인의 인사·회계·ERP 통합 관리 |
| **전신** | W4C (Welfare for Community, 사회복지시설정보시스템) → 2025.01.31 희망이음에 통합 |
| **주요 기능** | 인사관리, 회계관리, 전자결재, 법인 업무 관리 |
| **장기요양기관 관계** | 장기요양기관도 사용 대상이긴 하지만, HR/회계/ERP 기능만 해당 |
| **급여제공기록 관련** | 없음 (급여제공기록은 공단 별도 시스템) |
| **상태변화기록 관련** | 없음 (상태변화기록은 기관 자체 관리) |

**출처**: 희망이음 인사·회계기능(ERP) FAQ (kaccc.org, 2025.1.31 기준); 한국사회보장정보원 공지

## 2. "원장님이 방문 후 며칠 이내에 입력하는 것"의 정체

조사 결과, 원장님이 요양보호사 방문 후 처리하는 업무는 다음과 같다:

### 공단 시스템에서의 업무 (희망이음 아님)

1. **RFID 전송현황 확인**: 요양보호사가 스마트장기요양앱으로 전자태그를 찍고 서비스 내용을 전송하면, 기관관리자(원장)가 **장기요양기관 업무지원시스템(공단 웹)**에서 전일 전송 현황을 확인한다.

2. **미전송·오류건 보완**: 태그 미전송이나 오류가 있을 경우 급여제공기록지를 수기로 보완 작성하여 관리한다.

3. **월 단위 급여비용 청구**: 매월 급여제공기록을 기반으로 공단에 비용을 청구한다.

### 희망이음에서의 업무 (방문 기록과 무관)

- 직원 인사관리 (채용, 퇴직, 급여 등)
- 회계 처리 (수입, 지출, 결산 등)
- 전자결재
- 이 업무들은 **개별 방문 후 며칠 이내에 하는 것이 아니라**, 월간/분기별 행정 업무다.

### 결론

**"방문 후 며칠 이내에 원장님이 희망이음에 입력"이라는 설명은 정확하지 않다.** 방문 후 관리하는 것은 급여제공기록의 RFID 전송현황 확인이며, 이는 공단 시스템(장기요양기관 업무지원시스템)에서 이루어진다. 희망이음은 HR/회계용이다.

## 3. 보듬 사업 논리에 대한 시사점

### 보듬이 자동화하는 영역

| 보듬 기능 | 해당 서류 | 현재 전산 시스템 |
|:---|:---|:---|
| 음성→서술형 기록 자동 생성 | 상태변화기록지 | **없음** (수기/엑셀) |
| 보호자 알림톡 자동 발송 | 보호자 소통 | **없음** (개인 카톡) |

### 보듬이 건드리지 않는 영역

| 영역 | 해당 서류 | 담당 시스템 |
|:---|:---|:---|
| 수가 청구 | 급여제공기록지 | 스마트장기요양앱 + 공단 웹 |
| HR/회계 | 인사·회계 | 희망이음 ERP |

### 핵심 논거

1. **상태변화기록지에는 전산 시스템이 없다** — 이것이 보듬의 존재 이유. 공단 앱도 희망이음도 이 영역을 커버하지 않는다.

2. **급여제공기록지는 공단 앱이 이미 담당한다** — 보듬은 이를 대체하지 않으며, "기존 인프라 위에 얹히는 Add-on"이라는 사업 모델과 일치한다.

3. **희망이음 통합(2025.01.31)은 보듬과 무관하다** — 희망이음은 HR/회계 기능만 담당하며, 상태변화기록지 작성 기능을 추가한 적이 없다.

## 4. 발표 평가 예상 Q&A 방어

### Q. "급여제공기록지와 상태변화기록지가 뭐가 다릅니까?"

> "급여제공기록지는 수가 청구를 위한 법정 서식으로, 체크리스트 형태이며 스마트장기요양앱에서 전자 입력합니다. 상태변화기록지는 기관 평가를 위한 서술형 기록으로, 수급자의 신체·인지·정서 상태를 구체적으로 서술해야 하며, 별도 전산 시스템 없이 수기로 작성합니다. 보듬은 이 서술형 상태변화기록지를 음성 AI로 자동 생성합니다."

### Q. "희망이음이 통합됐는데 그걸로 해결 안 됩니까?"

> "희망이음은 기존 W4C(사회복지시설정보시스템)의 인사·회계·ERP 기능을 통합한 시스템입니다. 상태변화기록지 작성 기능은 W4C에도 없었고 희망이음에도 없습니다. 급여제공기록은 공단의 스마트장기요양앱이 담당하고, 상태변화기록은 여전히 전산 시스템 없이 수기로 관리됩니다. 보듬은 바로 이 빈 영역을 채웁니다."

### Q. "공단 앱이 리뉴얼됐는데 상태변화기록도 넣을 수 있지 않나요?"

> "공단 앱 리뉴얼(2025년)은 급여제공기록 서식을 3종에서 14종으로 확대한 것입니다. 이것은 수가 청구용 체크리스트이며, 기관 평가용 서술형 상태변화기록과는 완전히 다른 서류입니다. 오히려 리뉴얼로 입력 항목이 늘어 현장 부담이 가중됐고, 이는 보듬의 Zero-Typing 가치를 더 강화합니다."

## 5. 검증 수준 및 한계

| 항목 | 검증 수준 | 비고 |
|:---|:---|:---|
| 급여제공기록지 = 수가 청구용 법정 서식 | **팩트 확인** | 국가법령정보센터 고시 별표·서식으로 확인 |
| 상태변화기록지 = 기관 내부 보관, 주 1회 서술형 | **팩트 확인** | 다수의 방문요양 실무 가이드, 평가매뉴얼 검색 스니펫으로 확인 |
| 상태변화기록지에 전산 시스템 없음 | **팩트 확인** | 모든 출처에서 "수기 작성" 언급. 전산 입력 채널 존재하는 출처 없음 |
| 희망이음 = HR/회계/ERP 기능 | **팩트 확인** | 한국사회보장정보원 공지, 희망이음 FAQ으로 확인 |
| 희망이음에 상태변화기록 기능 없음 | **간접 확인** | 희망이음 기능 목록에 없음. 직접적인 "없다"는 출처는 없으나, 있다는 출처도 전무 |
| 원장님이 방문 후 확인하는 것 = 공단 RFID 전송현황 | **팩트 확인** | 케어포 RFID 가이드북, 스마트장기요양앱 기능 설명으로 확인 |
