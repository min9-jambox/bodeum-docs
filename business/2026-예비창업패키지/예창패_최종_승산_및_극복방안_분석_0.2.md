## 2. 0.2에서 개선된 것 — 잘 된 부분

### ✅ MVP 범위 명확한 1/2단계 분리
"6~8월 개발 → 9월 시범 → 10월 고도화"로 일정이 현실화됐다. 심사위원이 가장 불신하는 "1인이 모든 것을 동시에 만든다"는 느낌이 줄었다. 이 구조는 유지해야 한다.

### ✅ 프로모션 기간 1년 → 6개월
수익화 속도 우려를 해소했다. 지속가능성 관점에서 훨씬 현실적이다.

### ✅ 한 줄 소개 명확화
"요양보호사가 음성으로 1~2분 말하면, AI가 공단 제출용 관찰일지와 보호자 카카오 알림장을 동시에 자동 완성하는 B2B SaaS" — 심사관이 서비스를 바로 이해한다.

### ✅ 다국어 지원 + 녹음 시간 현실화
"10초"라는 비현실적 수치가 사라지고 "1~2분"으로 정직해졌다. 외국인 요양보호사 언급은 2024년 E-7 비자 신설이라는 실제 정책과 연결되어 타이밍이 좋다.

### ✅ 특허 2건 계획
단순 방어를 넘어 글로벌 규격 표준화라는 특허 2호가 추가됐다. 심사위원에게 "기술 기업"으로 읽힌다는 점에서 플러스다.

## 3. 여전히 치명적인 문제 5가지

### ❌ 문제 1: 인터뷰 공란 — 이것 하나로 탈락 가능 [최우선 해결]

현재 사업계획서 1-1 섹션에 **"이 부분을 반드시 채워 넣으세요"라는 공란이 그대로 제출될 위험이 있다.** 예창패 심사위원은 이 칸이 비어 있으면 "이 사람은 시장조사조차 안 했다"고 판단하고 탈락 처리한다.

**문제의 심각성**: 아이디어가 아무리 좋아도 수요 검증이 없으면 서류 단계에서 컷된다. 예창패는 "창업 준비가 된 사람"에게 주는 돈이지, "아이디어가 있는 사람"에게 주는 돈이 아니다.

**극복 방안**:
- 제출 전까지 원장님·요양보호사 최소 10건, 이상적으로는 15건 인터뷰
- 인터뷰에서 반드시 받아야 할 3가지 답변:
  - ① "지금 관찰일지 작성이 얼마나 힘든가?" (고통 확인)
  - ② "말로 하면 AI가 써준다면 쓸 의향이 있나?" (수요 확인)
  - ③ "월 1만 원에 결제할 의향이 있나?" (지불 의사 확인)
- 인터뷰 결과 중 긍정적인 것 5~7건을 실명(또는 이니셜)·날짜와 함께 1-1 섹션에 삽입

---

### ❌ 문제 2: 자문위원 이름 공란 — 팀 구성 점수 손실 [즉시 해결 가능]

4-1 섹션 자문위원 칸에 "○○○"이 그대로다. 이것도 채워 내지 않으면 심사위원에게 "구체적인 지원 조직이 없다"는 신호를 준다.

**#JMG 판단**: 친형(사회복지사 2급)을 자문위원으로 기재하는 것은 충분히 타당하다.
- 사회복지사 2급은 복지 서비스 전달 체계, 기관 운영 구조, 보호자 소통 프로토콜에 대한 전문 지식을 보유
- 요양보호사 현장 경험이 없다는 점은 오히려 솔직하게 기재하는 게 낫다 ("복지 제도·서비스 설계 자문 담당")
- 현장 경험 보유 자문위원은 별도로 1명만 더 확보하면 팀 구성이 충분히 보완됨

**극복 방안**: 자문위원 칸을 다음과 같이 채울 것

```
자문위원 1: 홍○○ (사회복지사 2급, 복지 제도·서비스 설계 자문)
자문위원 2: (현직 방문요양센터 원장님 또는 요양보호사 — 인터뷰 섭외 과정에서 확보 추진 중)
```

### ❌ 문제 3: 1단계 예산 구조 내부 모순

1단계 2,000만 원 예산 합계를 보면: Mac Studio 800 + API 500 + 자문비 400 + 특허 2건 600 = **2,300만 원**이다. 100만 원 초과 상태다. 심사위원이 예산표를 보는 순간 산술 오류가 잡힌다.

**극복 방안**: 자문비 400만 원 → 200만 원으로 줄이거나, 특허 1건을 2단계로 미루거나, API 예산을 400만 원으로 조정해 합계를 정확히 2,000만 원으로 맞춰야 한다. 제출 전 반드시 확인.

---

### ❌ 문제 4: 경쟁사 케어로그에 대한 과소평가

현재 계획서는 케어로그를 "양방향 채팅으로 감정노동 유지"라고 약점으로 서술하고 있다. 그러나 케어로그는 2025년 4월 **구글·중기부 창구 프로그램 7기에 100개사 중 1개**로 선정됐고, 음성·사진 기반 일지 자동생성 **특허를 이미 보유**하고 있다. 심사위원이 케어로그를 알고 있을 가능성이 높으며, "차별점이 충분한가?"라는 질문을 반드시 던질 것이다.

**극복 방안**: 케어로그 비교 서술을 다음과 같이 강화할 것
- 케어로그가 이미 선정되고 특허도 있다는 사실을 먼저 인정
- 그 위에서 "케어로그가 해결 못한 3가지"를 명확히 서술 (보호자 앱 설치 강제 / 채팅 = 감정노동 재생산 / UI 복잡도)
- 보듬의 특허 2건이 케어로그 특허와 겹치지 않는 이유(정합성 검증 레이어 차별화)를 명시

---

### ❌ 문제 5: 단위 경제성 BEP 수치 불일치

BEP 표에 "약 800개소 = 전국 재가기관의 3.5%"라고 나와 있는데, 실제 계산은 다음과 같다.

- 월 고정비 300,000원에서 "월 순수익 500만 원"을 달성하려면 필요 센터 수 = (5,000,000 + 300,000) ÷ 7,200 ≒ **736개소**
- 단순 BEP(고정비만 회수)는 300,000 ÷ 7,200 ≒ **42개소**

"약 800개소"는 어떤 가정에서 나온 수치인지 기준이 불명확하다. 심사위원이 계산해보면 맞지 않는다는 것을 즉시 안다.

**극복 방안**: BEP 표에 기준을 명확히 표기할 것
- 예: "월 순수익 500만 원 달성 기준 약 736개소 (= (5,300,000원 ÷ 7,200원), 전국 재가기관의 약 3.2%)"
- 또는 순수 손익분기점(고정비 회수)은 42개소, 대표 월급 200만 원 포함 시 약 320개소 등으로 단계별로 서술하는 것이 더 설득력 있음

## 4. 심사위원이 반드시 던질 질문 5가지 — 지금 답을 준비해야 한다

**Q1. "케어로그도 음성 기반인데, 어떻게 다릅니까?"**
- 현재 답변 준비도: △ (차별점은 있으나 케어로그 특허 존재를 적극 인정하지 않음)
- 보완: "케어로그는 특허가 있습니다. 그러나 그 특허는 음성→텍스트 변환 '프로세스' 특허입니다. 저희는 음성과 사진의 논리적 정합성 검증이라는 '신뢰도 검증 방법론' 특허로 비침해 회피가 가능합니다."

**Q2. "1인 개발자가 4개월 안에 iOS/Android 앱을 다 만들 수 있습니까?"**
- 현재 답변 준비도: △ (MVP 범위 분리로 개선됐으나 구체적 근거 부족)
- 보완: Claude Code 실사용 개발 속도 데이터, 기존 유사 프로젝트 완성 레퍼런스를 팀 역량 섹션에 1~2줄로 추가

**Q3. "원장님이 월 1만 원을 내겠다는 실제 증거가 있습니까?"**
- 현재 답변 준비도: ✗ (논리만 있고 증거 없음)
- 보완: 위 인터뷰 결과가 전부다. 이것 없이는 이 질문을 막을 수 없다.

**Q4. "예산이 2,000만 원인데 특허 2건에 600만 원을 쓰는 게 맞습니까?"**
- 현재 답변 준비도: △
- 보완: 특허가 경쟁사(케어로그) 진입 방어와 투자 유치의 핵심 자산임을 명확히 설명. 단, 예산 합계가 2,000만 원을 초과하지 않도록 조정 필수.

**Q5. "팀이 1명인데 사업화까지 할 수 있습니까?"**
- 현재 답변 준비도: △
- 보완: 1인 팀의 역설적 강점(의사결정 속도 + AI 코딩 에이전트 활용)을 구체적 수치로 표현. 예: "Claude Code 활용 시 동일 기능 구현에 기존 팀 대비 약 3배 빠른 이터레이션 경험 보유 (기존 프로젝트 OOO 사례)"

## 6. 최종 합격 가능성 시나리오

### 시나리오 A: 인터뷰 0건으로 현재 상태 제출
- 예상 점수: **80점**
- 서류 통과 확률: **40~50%**
- 최종 선정 확률: **10~15%**
- 결론: 논리는 완성됐지만 증거가 없어 최종 단계에서 탈락

### 시나리오 B: 인터뷰 10건 + 자문위원 2명 + 예산 수정 후 제출
- 예상 점수: **88~91점**
- 서류 통과 확률: **80~85%**
- 최종 선정 확률: **50~60%**
- 결론: 충분히 경쟁력 있는 계획서

### 시나리오 C: B + 케어로그 비교 논리 강화 + 발표 준비
- 예상 점수: **92~95점**
- 서류 통과 확률: **90%+**
- 최종 선정 확률: **65~75%**
- 결론: 이 수준이면 본선에서도 싸울 수 있다
