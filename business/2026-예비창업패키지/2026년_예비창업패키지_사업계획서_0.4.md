## ❶ 문제인식 (Problem)

### 1-1. 창업 아이템의 개발 동기 및 목적

#### 개발 배경 — 팩트 기반 현장 문제

2026년 현재 전국 노인장기요양보험 인정자 수는 **116만 5,000명**을 돌파하였으며, 재가급여(방문요양·방문목욕 등) 공단부담금은 전체의 62.6%인 9조 2,412억 원에 달한다. (국민건강보험공단 『2024 노인장기요양보험 통계연보』)

이 거대한 돌봄 시장의 현장을 떠받치는 전국 현직 요양보호사 약 66만 명(2024년 7월 기준) 중 **60세 이상 비율이 66.1%**에 달한다. 20·30대는 1% 미만으로, '노인이 노인을 돌보는' 구조가 고착되어 있다. (국민건강보험공단 NHIS 요양보호사 실태조사, 2024년 7월)

이 고령 현장 종사자들은 단 하나의 앱이 아닌, **세 개의 분리된 행정 채널**에 동일한 정보를 매일 중복 입력해야 하는 삼중 고통에 시달린다.

| 채널 | 목적 | 현장의 실제 고통 |
|:---|:---|:---|
| ① 건보공단 앱(스마트장기요양) | 수가 청구용 출퇴근 태그 + 전자서명 | 법적 필수. 대체 불가. |
| ② 개인 카카오톡 | 보호자 안심 소통 | 개인 폰 사생활 침해 + 24시간 감정노동 |
| ③ 수기 or 민간 ERP 타이핑 | 주 1회 서술형 관찰일지 (기관 평가 필수) | 60대에게 타자 입력은 극한 스트레스 |

> **[핵심 팩트] 왜 공단 앱만으로는 부족한가?**
> 건보공단 장기요양기관 평가 기준은 요양보호사에게 **"주 1회 이상 수급자의 신체·인지·정서 상태 변화를 구체적인 서술형으로 작성"**을 요구한다. 그러나 공단 앱은 O/X 체크 방식이므로, 요양보호사는 공단 앱 외에 반드시 민간 ERP에 별도로 타자 입력해야 하는 구조적 이중 작업이 발생한다.

이 구조적 문제로 인해 2028년에는 전국에서 **11만 6,000여 명의 요양보호사가 부족**할 것으로 전망된다. (건강보험연구원, 2023)

---

#### 현장 수요 검증 — 구조적 문제의 필연성

현장 수요는 개별 인터뷰 이전에 구조적으로 이미 증명된다. 아래 세 가지 팩트가 교차하면 수요는 필연적으로 존재한다.

**① 건보공단 평가 기준이 이중 작업을 강제한다 (정책 팩트)**
공단 앱(스마트장기요양)은 O/X 체크 방식이다. 그러나 장기요양기관 평가 매뉴얼은 "주 1회 이상 서술형 상태변화기록지 작성"을 의무화한다. 이 두 가지가 충돌하는 한, 요양보호사는 공단 앱 외에 별도 ERP나 수기 작성을 반드시 해야 한다. 이는 개인의 선택이 아닌 제도적 강제다. (국민건강보험공단 장기요양기관 평가 매뉴얼, 2024)

**② 종사자 66.1%가 60세 이상 — 디지털 입력 자체가 물리적 장벽 (인구 팩트)**
스마트폰 타자 입력에 가장 취약한 고령층이 이 시장 종사자의 3분의 2를 차지한다. 이들에게 ERP 타이핑은 단순한 불편이 아니라 실질적 진입 장벽이다. 해결책이 없는 상태에서 2028년 인력 부족 11만 6,000명 전망은 이 구조적 고통과 직결된다. (국민건강보험공단 NHIS 요양보호사 실태조사, 2024년 7월)

**③ 유사 방향의 스타트업이 이미 정부 지원을 받고 있다 (시장 검증 팩트)**
요양 현장 행정 자동화를 목표로 하는 스타트업이 2025년 구글·중소벤처기업부 창구 프로그램 및 중기부 R&D 과제를 통해 정부 지원을 받고 있다. 이는 이 시장의 수요가 이미 외부 기관에 의해 검증됐음을 보여주는 객관적인 증거다. 다만 기존 솔루션들은 공통적으로 보호자 전용 앱 설치를 강제하거나, 양방향 채팅 기능으로 오히려 요양보호사의 감정노동을 재생산하는 구조적 한계를 가진다. 보듬은 바로 이 공백에 집중한다.

### 1-2. 창업 아이템의 목적(필요성)

본 서비스 '보듬'은 요양 현장의 세 이해관계자 모두에게 가치를 제공한다.

**[요양보호사] Zero-Typing 해방**

복잡한 모바일 앱 타이핑을 완전히 배제한다. 퇴근 전 1~2분 음성 녹음(사투리·외국어 모두 지원)만으로 AI가 ① 평가 제출용 상태변화기록지(서술형 관찰일지)와 ② 보호자용 감성 알림톡을 동시에 완성한다. 현장 사진·동영상 첨부도 함께 가능하다. 개인 카톡 보호자 응대와 주간 수기 작성이 사라진다.

**[보호자] 무설치 안심 알림**

별도의 앱 설치 없이 전 국민이 이미 사용하는 카카오 알림톡으로 매일 부모님의 오늘 상태를 받아본다. 진입 장벽이 0이므로 열람률이 100%에 수렴한다.

**[원장님] 이중 작업 0% + 마케팅 무기**

보듬이 생성한 데이터를 기존 ERP(이지케어 등) 호환 엑셀로 원클릭 다운로드. 새로운 시스템을 배울 필요가 없다. 동시에 "AI 알림장을 보내주는 센터"라는 차별화로 신규 수급자 유치에 활용한다.

## ❷ 실현가능성 (Solution) — 창업 아이템 개발 계획

### 2-1. 창업 아이템의 개발·사업화 전략

#### 핵심 기술 스택: 린(Lean) AI 아키텍처

자체 AI 모델 개발을 배제한다. 세계 최고 수준의 상용 API를 요양 버티컬 도메인에 맞게 조립하는 방식으로 원가를 최소화하고 개발 속도를 극대화한다.

| 기술 영역 | 채택 기술 | 선택 근거 |
|:---|:---|:---|
| 음성 인식(STT) | OpenAI Whisper API | 사투리·다국어 인식, 현장 잡음 환경 대응 |
| 텍스트 생성 | GPT-4o-mini | 요양보호사 1인당 월 AI 원가 500원 이하 |
| 보호자 알림 | 카카오 알림톡 API | 건당 9~15원, 보호자 무설치 |
| 위치 감지 | GPS 지오펜싱 | 반경 50m 진입·이탈, 절전 모드 |
| 아키텍처 | RAG (검색 증강 생성) | W4C 표준 서식을 LLM 컨텍스트에 주입 |

#### MVP 개발 계획: 범위를 좁혀 확실히 완성한다

**⚠️ 핵심 원칙**: 1단계(2,000만 원 기간)에서는 **'음성→알림톡 단일 파이프라인 완성 + 5개 센터 시범 운영'만** 목표로 한다. 기능을 추가하는 것보다 핵심 파이프라인을 현장에서 실제 동작하는 수준까지 완성하는 것이 최우선이며, 2단계 자금은 이 실증을 바탕으로 집행한다.

**1단계 MVP (협약 후 약 5개월, 2026년 4~8월 개발 + 9월 시범 운영)**

| 기능 | 내용 | 완성 기준 |
|:---|:---|:---|
| 요양보호사 모바일 앱 | React Native, 단일 녹음 버튼 화면 | iOS/Android 실제 설치 및 음성 녹음 가능 |
| AI 파이프라인 | 음성→Whisper→GPT-4o-mini→알림톡 | 실제 발송 성공률 95%+ |
| 원장님 웹 백오피스 | 엑셀 업로드 + 일지 확인 + 엑셀 다운로드 | 원클릭 다운로드 동작 |
| 시범 운영 | 수도권 5개 센터 현장 테스트 (9월) | 요양보호사 만족도 조사 완료 |

> **[1인 개발 가능 근거]** 대표자는 25년 경력의 풀스택 아키텍트로, Claude Code·Cursor 등 AI 코딩 에이전트를 실무에 적극 활용하고 있다. 1단계 MVP는 기존 상용 API(Whisper·GPT-4o-mini·카카오 알림톡)의 조립에 집중하므로 AI 원천 기술 개발 없이 5개월 내 완성 가능하다.

**2단계 (2,000만 원, 중간평가 통과 후)**

- GPS 지오펜싱 출퇴근 넛지
- 프리미엄 마케팅 리포트 (원장님 직인 + 사진 갤러리)
- 멀티모달 정합성 검증 엔진 (음성-사진 교차 검증, 특허 연계)
- 100개소 확장 영업

#### 이중 작업(Double Entry) 우려 완전 차단 설계

> "어차피 공단 앱 써야 하는데, 앱이 하나 더 늘어나는 것 아닙니까?"

보듬이 대체하는 것은 공단 앱(법적 필수, 절대 대체 불가)이 아니다. 공단 앱 **'이외에'** 강요받는 두 가지를 소멸시키는 것이다.

- 삭제: 개인 카톡으로 보호자에게 장문 응대하는 감정 노동
- 삭제: 주 1회 수기 or ERP 타이핑 관찰일지 작성

결과적으로 삼중 작업(공단 앱 + 카톡 + 수기) → 단순화(공단 앱 + 보듬 1~2분). 현장에서 실제 녹음에 30초~1분, 사진·영상 첨부에 30초 내외가 소요되지만, 이는 기존 주 1회 수기 타이핑(평균 20~30분)과 매일 카톡 보호자 응대를 완전히 대체한다. **총 행정 부담이 현저히 줄어든다.** 음성 녹음뿐 아니라 현장 사진 및 짧은 동영상 첨부 기능도 제공하여 보호자에게 더욱 신뢰도 높은 돌봄 기록을 전달한다.

#### 시장 규모

| 구분 | 규모 | 근거 |
|:---|:---|:---|

| SOM 1차년도 목표 | 100개소 | 예창패 자금 프로모션 타겟 |

#### 경쟁사 분석 및 차별화

현재 요양 행정 시장에는 크게 두 종류의 플레이어가 있다. 하나는 **청구 행정 중심 ERP**(이지케어텍, 하이케어 등)이고, 다른 하나는 **소통·일지 자동화 솔루션**들이다. 두 유형 모두 현장 요양보호사의 '입력 행위 자체'를 줄여주지 못한다는 공통된 한계를 갖는다.

| 구분 | 강점 | 공통 한계 | 보듬과의 관계 |
|:---|:---|:---|:---|
| 청구 행정 ERP (이지케어텍 등) | 청구 행정 자동화, 높은 시장 점유율 | 요양보호사 수동 타이핑 강제, 프론트엔드 개선 없음 | **공존 (Add-on)** |
| 소통·일지 솔루션 (기존 다수) | 일지 작성 보조, 보호자 소통 채널 제공 | 보호자 전용 앱 설치 강제, 양방향 채팅으로 감정노동 재생산, 타이핑 여전히 필요 | **차별화 대상** |

보듬은 두 유형 모두가 해결하지 못한 **프론트엔드 공백 — 60대 요양보호사의 타이핑 행위 자체를 소멸시키는 것** — 에 집중한다.

**[보듬만의 결정적 차별화 3가지]**

1. **입력 방식**: 기존 모든 솔루션 → 수동 타이핑 강제 / 보듬 → 음성 1~2분 (타이핑 0)
2. **보호자 채널**: 기존 소통 솔루션 → 전용 앱 설치 필요 / 보듬 → 카카오 알림톡 단방향 무설치 (열람 허들 0)
3. **ERP 관계**: 기존 모든 솔루션 → 독립 생태계 구축 / 보듬 → 기존 ERP 위에 얹히는 가벼운 Add-on (도입 저항 0)

**[보듬 특허 전략 — 독자 기술 영역 선점]**

- **특허 1호 (방어)**: 음성과 사진 간 논리적 정합성을 검증하는 신뢰도 측정 방법론 — 기존 솔루션에 없는 "AI 기록 오류 검증" 영역
- **특허 2호 (확장)**: 다국어(베트남어·몽골어 등) 음성을 한국 W4C 표준 서식으로 자동 변환하는 이기종 규격 매핑 기술 — 외국인 요양보호사 증가(E-7 비자 신설, 2024년 7월) 정책과 연동되는 독자 영역

### 2-3. 정부지원사업비 집행 계획

## ❸ 성장전략 (Scale-up) — 사업화 추진 전략

### 3-1. 경쟁제품·경쟁사 분석 및 목표 시장 진입 전략

#### 시장 진입 전략: 트로이 목마

대형 ERP와 정면충돌하지 않는다. "기존 이지케어 그대로 쓰시면서, AI 알림장만 1만 원에 추가하세요"라는 가벼운 플러그인으로 침투한다.

**원장님 영업 소구점 3가지:**

1. **요양보호사 이직 방지 보험**: "월 1만 원이 요양보호사 1명 퇴사로 날아가는 매출 손실보다 압도적으로 싸다."
2. **수가 환수 방어 컴플라이언스**: "LLM이 공단 평가 기준에 맞춰 서술형 일지 자동 생성. 부실 기재로 인한 환수 조치 리스크를 차단한다."
3. **신규 수급자 유치 마케팅**: "'저희 센터는 AI가 매일 어르신 안부를 카톡으로 보내드립니다' — 보호자 상담에서 즉각적인 차별화 포인트."

**초기 집중 타겟 3종:**

- **소규모 방문요양 센터 (20인 이하)**: 행정 전담 없이 원장이 모두 감당. 가장 높은 Pain Point.
- **방문목욕 서비스**: 젖은 손으로 타이핑 물리적 불가. Zero-Typing이 가장 명백하게 필요한 환경.
- **개원 1년 차 이내 신설 기관**: Switching Cost 0. 초기 표준 도구로 자리잡기 가장 쉬운 타겟.

### 3-2. 비즈니스 모델(수익화 모델) 및 투자유치 전략

#### 과금 구조

| 단계 | 요금 | 내용 |
|:---|:---|:---|
| 도입 첫 달 | 무료 (알림톡 포함) | 예창패 광고선전비 예산 활용 |
| 프로모션 (6개월) | 월 5,000원 | 초기 가입 센터 락인 |
| 정상화 | 월 10,000원 (20인 이하 기준) | 초과 10명당 10,000원 추가 |
| 알림톡 발송비 | 실비 종량제 (센터 직접 충전) | 회사 변동비 리스크 0 |

**보호자는 무과금 원칙**: 이미 건보 본인부담금을 내는 보호자에게 추가 구독료를 요구하지 않는다.

#### 단위 경제성(Unit Economics)

| 항목 | 수치 | 산출 근거 |
|:---|:---|:---|
| 센터 1곳 월 매출 | 10,000원 | 기본 구독료 |
| 알림톡 발송비 (200건/월) | ~1,800원 | 건당 약 9원 × 200건 |
| LLM API 비용 | ~200원 | GPT-4o-mini 경량 모델 |
| 서버 종량제 | ~500원 | AWS 클라우드 |
| **센터 1곳 공헌이익** | **~7,200원** | **마진율 약 72%** |
| 월 고정비 | ~300,000원 | 서버 기본료·도메인·예비비 |
| **손익분기점 (고정비 회수)** | **약 42개소** | 300,000 ÷ 7,200 |
| **대표 월급 200만 원 포함 BEP** | **약 320개소** | 2,300,000 ÷ 7,200 |
| **월 순수익 500만 원 달성 목표** | **약 736개소** | 5,300,000 ÷ 7,200 = 전국 재가기관의 약 3.2% |

전국 재가기관 22,735개소 중 **단 3.2%**만 확보하면 월 순수익 500만 원 자동화 수익 달성. 이직률 감소 효과로 사회적 가치와 수익이 동시에 발생하는 구조다.

#### 투자유치 로드맵

- **Phase 1 (2026)**: 예창패 자금 → 100개소 유료 레퍼런스 + 살아있는 Traction 증명
- **Phase 2 (2027)**: 소셜벤처 인증 → ESG/임팩트 펀드 Seed/Pre-A 3~5억 원
- **Phase 3 (2028~)**: 공단 청구 ERP 모듈 오픈 → 이지케어 고객 흡수 → 트로이 목마 완성

## ❹ 팀 구성 (Team) — 대표자 및 팀원 구성 계획

### 4-1. 대표자의 보유 역량

#### 25년 차 풀스택 시스템 아키텍트

기획→클라우드 아키텍처→프론트엔드→백엔드 전 과정을 1인이 통제. 외주 의존 없이 예창패 지원금을 100% 인프라와 마케팅에 집중 투하 가능.

**기술 역량:**
- 클라우드: AWS (Lambda, ECS, RDS, S3), 오토스케일링 설계
- 백엔드: Spring Cloud, 마이크로서비스 아키텍처
- 프론트엔드: React, React Native (iOS/Android 크로스 플랫폼)
- AI/LLM: OpenAI API, Claude API, RAG 파이프라인, 프롬프트 엔지니어링

**1인 개발 체제의 강점:**
- Claude Code 등 AI 코딩 에이전트를 실무 파트너로 활용 → 개발 속도 레거시 팀(4~5인) 대비 약 3배 빠른 이터레이션 가능
- 의사결정 지연 없이 시장 피드백에 즉각 대응 가능
- 외주 의존으로 인한 예산 낭비 리스크 0 → 지원금 전액을 인프라·마케팅에 투입
