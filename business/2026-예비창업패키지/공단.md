# 공단 앱(스마트장기요양) 완전 해부 — 보듬 제품 전략 근거 문서
> 작성일: 2026년 3월
> 목적: 스마트장기요양 앱의 실제 구조, 요양보호사 현장 업무 흐름, 공단 연동 가능성을 정확히 파악하여 보듬의 포지셔닝 근거로 활용

---

## 1. 공단 앱을 직접 볼 수 있나?

### 다운로드 방법
- **안드로이드**: 구글 플레이스토어에서 "스마트장기요양" 검색 후 설치 가능
- **아이폰**: 앱스토어에 정식 등록되어 있지 않음. 공단 홈페이지(longtermcare.or.kr) 하단 QR코드 촬영 방식으로만 설치 가능

### 실제 사용 가능 여부: **불가능**
앱은 설치할 수 있지만, **2025년 리뉴얼 이후 건강보험·공동·금융 인증서 기반 로그인으로 변경**됐다.
요양보호사·수급자·보호자로 등록된 실제 자격자가 아니면 앱 내부 화면을 볼 수 없다.

> 💡 **대안**: 앱스토어 리뷰 이미지, 현장 원장님·요양보호사 인터뷰 시 직접 화면 확인 요청

## 2. 요양보호사 하루 업무 전체 흐름

### 공단 앱(스마트장기요양) 관련 행위

```
[수급자 댁 도착]
    ↓
① 시작 태그 전송
   - 수급자 댁에 부착된 NFC/블루투스 태그에 스마트폰 접촉
   - 서비스 시작 시간이 공단 서버에 실시간 전송 → 수가 청구 근거 생성
   - 수급자·보호자 스마트폰에 "선생님이 도착했습니다" Push 알림 자동 발송

[서비스 제공 중: 2~4시간]
    ↓
② 돌봄 서비스 제공
   - 신체지원: 식사 보조, 세면, 옷 갈아입기, 이동 보조, 화장실 도움, 기저귀 교체
   - 가사지원: 청소, 빨래, 식사 준비, 설거지, 외출 동행
   - 인지·정서지원: 대화, 산책 동행, 인지 훈련

[서비스 종료 직전]
    ↓
③ 급여제공기록 입력 (공단 앱 내 — 요양보호사가 직접 입력)
   - 2025년 리뉴얼로 기존 3종 → 14종 서식으로 확대됨
   - 오늘 제공한 서비스 항목 선택 + 어르신 상태 간략 서술
   - ⚠️ 아이러니하게도 리뉴얼로 입력할 항목이 더 많아졌다
   - 앱스토어 리뷰: "대부분 요양보호사 연령대가 60·70대인데 다들 너무 힘들어 하신다,
     오셨다가 그냥 가시는 분이 대부분" / "사용하라고 만든 앱인지 의문"

④ 전자서명 받기 (공단 앱 내)
   - 수급자 또는 보호자의 전자서명을 앱 화면에서 직접 받아야 함
   - 치매·거동 불편 어르신에게 스마트폰 서명 요청이 현실적으로 매우 어렵다는 현장 불만 多
   - ⚠️ 전자서명은 급여제공기록 입력 이후에 받는 구조
     → 내용이 없는 상태에서 서명을 먼저 받는 것은 불가능

[수급자 댁 출발]
    ↓
⑤ 종료 태그 전송
   - 다시 태그에 접촉, 서비스 종료 시간 공단 서버에 전송
   - 수급자·보호자에게 "서비스가 완료됐습니다" Push 알림 자동 발송
   - ⚠️ 종료 태그 이후에는 해당 회차의 기록 수정이 원칙적으로 불가
     → 급여제공기록 입력과 전자서명은 반드시 종료 태그 이전에 완료해야 함
```

### ⚠️ 공단 앱 14종 서식의 실제 내용

"14종"은 급여 유형별 서식의 수다 (방문요양용 1개, 방문목욕용 1개, 방문간호용 1개 등).
방문요양 서식 안에 들어가는 **실제 입력 항목**은 아래와 같다.

**[신체활동지원]**
세면·구강관리 / 몸단장(머리빗기·면도) / 옷 갈아입히기 / 목욕 도움 /
식사 도움 / 체위 변경·이동 도움 / 신체기능 유지·증진 / 화장실·배설 도움

**[일상생활지원(가사활동)]**
청소·정리정돈 / 세탁 / 식사 준비·설거지 / 외출 동행 / 일상업무 대행

**[인지·정서지원]**
인지활동 지원 / 말벗·의사소통 도움

**[공통 기록 항목]**
방문 시작·종료 시간 / 어르신 당일 상태(건강·기분·식사량·배변) /
특이사항 / 요양보호사 서명 + **수급자(보호자) 전자서명**

> 💡 **핵심 포인트**: 어르신 당일 상태(건강·기분·식사량)와 특이사항은 여전히 요양보호사가 직접 타이핑해야 한다. 이 서술 부분이 60대에게 가장 힘든 구간이다.

## 3. 공단 앱 밖의 별도 업무 — 보듬이 타겟하는 영역

### ③ 상태변화기록지 (주 1회 이상, 기관 평가 대비 필수)

| 항목 | 내용 |
|:---|:---|
| **작성 주기** | 주 1회 이상 (최소 주 1회, 권장 2~3일 1회) |
| **작성 방법** | 수기 작성 또는 민간 ERP 타이핑 |
| **작성 주체** | 요양보호사 본인 |
| **제출처** | **공단 웹·앱에 제출하는 것이 아님** |
| **보관처** | 기관 내 보관 → 공단 정기평가(3년 주기) 실사 시 확인 |
| **법적 근거** | 장기요양기관 평가 기준 29번 항목 "주 1회 이상 서술형 상태변화기록지 작성 충실도" |

> ✅ **"주 1회 공단 웹에 입력"은 틀린 표현이다.**
> 정확한 표현: "주 1회 이상 기관 내에서 작성·보관하는 서술형 기록"

서술 내용 예시:
- "기침 약간 있었으나 식사 잘하심"
- "왼쪽 무릎 통증 호소, 이동 시 부축 제공"
- "오전 중 말벗 15분 진행, 가족 이야기 즐겁게 나눔"

### ④ 보호자 카카오톡 응대 (매일, 비공식 감정노동)

공단 앱의 Push 알림은 "서비스 시작·종료"라는 사무적 알림만 가능하다.
"오늘 어머니 식사 잘 하셨고 기분 좋으셨어요 😊" 같은 감성 소통은
**요양보호사 개인 카카오톡으로 따로 보내야 한다.**
보호자가 회신하면 퇴근 후에도 응대해야 하는 24시간 감정노동으로 이어진다.

## 4. 공단 앱 → 보듬 자동 연동 가능성

### 결론: 현시점 직접 자동화 불가능. 단, 우회 경로 있음.

### 왜 불가능한가?

공단이 외부 앱을 위한 **급여제공기록 '쓰기(Write)' API를 공개하지 않았기 때문.**
공공데이터포털(data.go.kr)의 건보공단 API는 전부 **읽기(Read) 전용** (기관 정보 조회 등).
외부에서 급여제공기록을 대신 써주는(Write) API는 공식적으로 존재하지 않는다.

### 하이케어 방식 (스크래핑) — 따라가면 안 되는 이유

하이케어 등 일부 민간 ERP가 "공단 롱텀 자동 연동"을 제공하는 방식은
공인인증서를 위임받아 공단 시스템에 **자동 접속(RPA/스크래핑)하는 구조**로 추정된다.

문제점:
1. **법적 리스크**: 공단이 명시적으로 허용하지 않은 자동 접속
2. **유지보수 지옥**: 공단이 UI를 조금만 바꿔도 연동 로직 전체가 깨짐 → 2025년 리뉴얼 시 실제로 현장 오류 폭발
3. **보안 문제**: 요양보호사 개인 인증서를 민간 앱에 위탁하는 구조

## 5. 보듬의 단계별 접근 전략

### 1단계 (MVP, 2026년): 공단 앱과 분리해서 가치 증명

공단 앱 영역은 건드리지 않는다. 공단 앱이 **전혀 해결하지 못하는 두 가지**만 없앤다.

| 현재 고통 | 보듬의 해결 |
|:---|:---|
| 주 1회 이상 수기 상태변화기록지 | 음성 1~2분 → AI 서술형 자동 생성 → PDF/엑셀 기관 보관 |
| 매일 개인 카톡 보호자 응대 | 알림톡 자동 발송 → 보호자 무설치 수신 |

**이것만으로도 충분히 강력하다.** 공단 API 없이 즉시 구현 가능하고, 상태변화기록지는 공단 웹에 제출할 필요도 없기 때문에 연동 이슈 자체가 없다.

### 2단계 (2027년~): 공단 파트너십 협력 추진

- 소셜벤처 인증 후 공단 디지털 혁신 파트너 트랙 등록 추진
- 공단이 자체 앱 UX 불만(앱스토어 리뷰 폭발)을 인식하고 있어 외부 솔루션 파트너십 필요성이 커지는 시점
- 공단 읽기 전용 API(서비스 계획 정보)를 활용해 음성 입력과 계획 항목을 자동 매칭하는 보조 기능 개발

### 3단계 (2028년~): 공단 앱 대체 — 트로이 목마 완성

- 공단이 공식 API를 개방하거나 보듬과 협약을 맺으면, 보듬에서 음성 입력 → 급여제공기록 자동 생성 → 공단 서버 전송까지 원스톱 처리
- 공단 앱을 열 필요 자체가 없어지는 구조
- 이 단계에서 이지케어 등 레거시 ERP 고객을 본격적으로 흡수 시작

## 6. 심사위원 예상 공격과 방어 논리

### Q. "공단 앱 입력은 여전히 해야 하잖아요. 보듬을 써도 입력이 완전히 없어지는 건 아니잖아요?"

> "맞습니다. 공단 앱은 수가 청구를 위한 법적 필수 도구로, 저희가 대체하지 않습니다.
> 보듬이 없애주는 것은 공단 앱이 담당하는 영역이 아닙니다.
>
> 공단 앱 이후에도 요양보호사에게 남아있는 두 가지 비공식 고통이 있습니다.
> 하나는 주 1회 이상 수기로 작성해야 하는 상태변화기록지이고,
> 다른 하나는 퇴근 후에도 개인 카카오톡으로 보호자를 응대해야 하는 감정노동입니다.
>
> 이 두 가지가 요양보호사들이 '일이 끝나도 일이 끝나지 않는다'고 느끼는 본체입니다.
> 보듬은 바로 이것을 소멸시킵니다."

### Q. "상태변화기록지를 AI로 만들면 공단에서 인정해 주나요?"

> "상태변화기록지는 공단에 주기적으로 제출하는 서류가 아닙니다.
> 기관 내에서 작성·보관하다가 3년 주기 공단 정기평가 때 실사 대상이 되는 서류입니다.
> 보듬이 AI로 생성한 서술형 기록을 PDF로 출력하거나 ERP에 붙여넣기 하면
> 기관 내 보관 의무는 완전히 충족됩니다.
> 공단 API 연동이 필요 없는 영역이기 때문에, 1단계에서 즉시 구현이 가능합니다."

## 8. 공단 앱 서술 입력 자동화 — 앱 전환 없이 해결하는 방법

> 이 섹션은 "보듬에서 생성한 문구를 공단 앱에 앱 전환 없이 자동으로 입력할 수 없을까?"라는 질문에 대한 기술·정책 분석이다.

### 문제 정의

공단 앱 급여제공기록의 서술 입력창(어르신 상태, 특이사항)은 여전히 요양보호사가 직접 타이핑해야 한다. 이를 해결하는 방법으로 "보듬 → 클립보드 복사 → 공단 앱에 붙여넣기"가 거론됐으나, **60대 요양보호사에게 앱 전환 3번은 현실적으로 큰 장벽**이다.

### 안드로이드: AccessibilityService — 기술적으로 완전 해결 가능

#### 작동 방식

안드로이드의 AccessibilityService는 다른 앱 위에 오버레이를 띄우고, 다른 앱의 입력창에 직접 텍스트를 삽입할 수 있는 시스템 API다. 구체적으로 세 가지 기능을 조합한다.

- **TYPE_ACCESSIBILITY_OVERLAY**: 공단 앱 실행 중에도 보듬 플로팅 버튼을 화면 위에 표시
- **onAccessibilityEvent**: 요양보호사가 공단 앱의 서술 입력창을 터치하는 순간 감지
- **performAction(ACTION_SET_TEXT)**: AI가 생성한 문구를 공단 앱 입력창에 직접 삽입 (붙여넣기조차 불필요)

#### 구현 시 요양보호사 경험

```
공단 앱에서 평소처럼 작업
    ↓
서술 입력창 도달 시
→ 화면 위에 보듬 플로팅 버튼 자동 출현
   [🎤 보듬으로 입력하기]
    ↓
버튼 터치 → 앱 전환 없이 그 자리에서 음성 팝업
    ↓
사투리로 10초 말하기
    ↓
AI 문구 생성 → 공단 앱 입력창에 자동 삽입 완료
    ↓
보호자 전자서명 → 종료 태그 (평소처럼)
```

**앱 전환 0번.** 공단 앱 안에서 모든 것이 완결된다.

#### ⚠️ 구글 플레이 정책 팩트체크 결과 — 조건부 가능

구글 플레이 공식 정책(AccessibilityService API 사용 정책, 2025년 기준)을 직접 확인한 결과:

> "Google Play에서는 다양한 애플리케이션에 AccessibilityService API를 사용할 수 있습니다. 그러나 장애인이 기기에 액세스할 수 있도록 지원하거나 장애로 인한 문제를 극복할 수 있도록 설계된 서비스만 접근성 도구라고 선언할 수 있습니다."
> (구글 플레이 정책 센터, AccessibilityService API 사용)

**핵심 판단:**

| 조건 | 보듬의 경우 |
|:---|:---|
| AccessibilityService API 사용 자체 | ✅ 허용 |
| 접근성 도구(isAccessibilityTool=true)로 선언 | ⚠️ 조건 충족 여부 검토 필요 |
| 비접근성 도구로 사용 시 | ✅ 가능하나 명시적 공개 및 동의 요건 충족 필요 |

구글이 제시하는 접근성 도구 예시 중 **"운동 장애가 있는 사용자를 지원하는 음성 기반 입력 시스템"** 이 있다. 60대 요양보호사의 디지털 입력 어려움은 이 범주에 해당한다고 충분히 주장할 수 있다.

**결론**: 구글 플레이 배포가 가능하다. 단, 두 가지 경로 중 하나를 선택해야 한다.
- **경로 A**: 접근성 도구로 선언 → Play Console에서 접근성 기능 설명 및 심사 통과 필요
- **경로 B**: 일반 앱으로 배포 → 앱 내에서 사용자에게 AccessibilityService 사용 목적을 명시적으로 공개하고 동의를 받는 절차 필요

둘 다 배포 불가 사유가 아니며, 적절한 선언과 공개를 통해 정상 배포 가능하다.

#### 추가 주의사항

- **최초 1회 권한 설정 필요**: 설정 → 접근성 → 보듬 → 활성화. 60대 요양보호사에게 어려울 수 있으므로 원장님이 설치 시 대신 설정해주는 온보딩 프로세스 필요.
- **공단 앱 UI 변경 시 유지보수 필요**: 입력창 위치가 바뀌면 감지 로직 업데이트 필요. 하이케어 스크래핑보다 훨씬 가볍고 안정적이나 제로는 아님.

### 아이폰(iOS): 세 가지 방법, 현실적 최선은 키보드 익스텐션

iOS는 안드로이드와 달리 다른 앱 위에 오버레이를 띄우거나 직접 텍스트를 삽입하는 것이 **애플 샌드박스 정책상 불가능**하다. 단, 세 가지 우회 방법이 있다.

#### 방법 A: 클립보드 복사 + 붙여넣기 (가장 단순, 그러나 앱 전환 필요)

보듬에서 생성한 문구를 클립보드에 자동 복사 → 요양보호사가 공단 앱 입력창에서 길게 눌러 붙여넣기.

- 구현 난이도: 매우 낮음 (Clipboard API 두 줄)
- 요양보호사 경험: 앱 전환 1회 + 붙여넣기 동작 필요
- **60대 내국인 요양보호사에게는 여전히 장벽. 단, 20~30대 외국인 요양보호사에게는 충분히 무난한 방법**

#### 방법 B: iOS 커스텀 키보드 익스텐션 (가장 현실적)

iOS는 iOS 8부터 서드파티 키보드를 시스템 전체에 적용할 수 있는 **Custom Keyboard Extension**을 공식 지원한다. 보듬 키보드를 iOS에 등록하면, 공단 앱 입력창에서도 보듬 키보드를 호출할 수 있다.

작동 방식:
1. 요양보호사가 최초 1회 설정 → 일반 → 키보드 → 키보드 추가 → "보듬 키보드" 선택
2. 공단 앱 서술 입력창 터치 시, 지구본 아이콘을 눌러 "보듬 키보드"로 전환
3. 보듬 키보드 화면에서 음성 녹음 버튼 터치 → 10초 말하기
4. AI가 생성한 문구가 공단 앱 입력창에 바로 입력됨

- 구현 난이도: 중간 (iOS Keyboard Extension 개발 필요)
- 요양보호사 경험: 앱 전환 없음. 키보드 전환 1번만 필요
- **안드로이드 AccessibilityService보다는 한 단계 불편하나, 클립보드 방식보다 훨씬 자연스럽다**
- **단점**: 최초 설정이 다소 복잡 (키보드 추가 과정). 역시 원장님이 설치 시 대신 설정해주는 온보딩 필요.

#### 방법 C: iOS 기본 받아쓰기 활용 (추가 개발 불필요)

iOS에는 키보드의 마이크 버튼을 누르면 모든 앱 입력창에서 음성으로 텍스트를 받아쓰는 기능이 기본 탑재되어 있다. 단, 이것은 iOS 기본 STT로 요양 도메인 최적화가 없고, 사투리 인식률이 낮으며, AI 문구 생성이 아니라 단순 음성→텍스트 변환만 가능하다.

- **보듬의 AI 정제(요양 서식에 맞는 서술형 변환)가 없으므로 단독으로는 불충분**
- 단, 추후 보듬 키보드 익스텐션과 결합하면 시너지 가능

---

### 플랫폼별 전략 요약

| 구분 | 방법 | 앱 전환 횟수 | 구현 난이도 | 주요 대상 |
|:---|:---|:---:|:---:|:---|
| **안드로이드** | AccessibilityService 오버레이 | **0번** | 중간 | 60대 내국인 요양보호사 (주력) |
| **iOS** | 커스텀 키보드 익스텐션 | **0번** (키보드 전환 1회) | 중간~높음 | 20~30대 외국인 요양보호사 |
| **iOS (단순)** | 클립보드 복사 + 붙여넣기 | **1번** | 낮음 | 외국인 요양보호사 (임시 방편) |

**MVP 1단계 권고**: 안드로이드 AccessibilityService 우선 개발. 현장 요양보호사 대부분이 안드로이드 사용자이며, 기술적으로도 정책적으로도 가장 명확한 경로다. iOS 키보드 익스텐션은 2단계에서 추가 개발한다.

AccessibilityService 구현이 완성되면, 보듬의 메시지가 격상된다.

| 상태변화기록지 | "수기 작성을 없앱니다" | 동일 |
| 보호자 카톡 | "감정노동을 없앱니다" | 동일 |
| **전체 포지셔닝** | "공단 앱 이외의 고통을 없앤다" | **"요양보호사의 모든 타이핑을 없앤다 — 진정한 Zero-Typing"** |

## 9. 팩트체크 최종 결론 요약

> 이 섹션은 "안드로이드 정책 이슈는 없는지", "아이폰은 더 좋은 방법이 없는지"에 대한 대화 내용을 정리한 최종 결론이다.

### 안드로이드 — 배포 가능, 두 가지 경로 중 선택

구글 플레이 공식 정책 원문 확인 결과, AccessibilityService 사용 자체는 금지되지 않는다. 아래 두 경로 중 하나를 명확히 선택해서 진행하면 된다.

**경로 A — 접근성 도구로 선언**

구글이 예시로 제시하는 접근성 도구 중 "운동 장애가 있는 사용자를 위한 음성 기반 입력 시스템"이 있다. 60대 요양보호사의 디지털 입력 어려움은 이 범주에 해당한다고 충분히 주장할 수 있다. Play Console에서 접근성 기능 사용 목적을 기술하고 심사를 통과해야 한다. 심사 통과 시 제약 없이 배포 가능하다.

**경로 B — 일반 앱으로 배포**

접근성 도구 선언 없이 일반 앱으로 배포하는 방법이다. 이 경우 앱 내에서 사용자에게 AccessibilityService 사용 목적을 명시적으로 안내하고 동의를 받는 화면을 제공하면 된다. 배포 자체에는 문제없다.

**공통 주의사항**

- 최초 1회 접근성 권한 설정 필요 (설정 → 접근성 → 보듬 → 활성화)
- 60대 요양보호사에게 이 설정이 어려울 수 있으므로, 원장님이 설치 시 대신 설정해주는 온보딩 프로세스를 서비스 표준으로 만들어야 함
- 공단 앱 UI 변경 시 입력창 감지 로직 업데이트 필요 (주기적 유지보수 항목으로 관리)

---

### iOS — 앱 전환 없는 방법 있음, 단계적 접근 권고

iOS는 안드로이드처럼 다른 앱 위에 오버레이를 띄울 수 없지만, 앱 전환 없이 해결하는 방법이 존재한다.

**최선: 커스텀 키보드 익스텐션 (iOS 8 이상 공식 지원)**

iOS가 공식 제공하는 Custom Keyboard Extension을 사용하면, 공단 앱 입력창에서도 보듬 키보드를 불러올 수 있다. 키보드 전환 1번만으로 음성 녹음 → AI 문구 삽입이 그 자리에서 완결된다.

- 앱 전환 없음. 키보드 전환 1번만 필요
- 최초 1회 설정 필요 (키보드 추가). 원장님이 온보딩 시 대신 설정
- 안드로이드보다 한 단계 불편하나, 클립보드 방식보다 훨씬 자연스러움

**차선: 클립보드 복사 + 붙여넣기**

보듬에서 문구 생성 후 클립보드 자동 복사 → 공단 앱 입력창에서 길게 눌러 붙여넣기.

- 앱 전환 1회 필요
- 구현 가장 단순 (코드 두 줄)
- 20~30대 외국인 요양보호사에게는 충분히 무난한 방법
- 60대 내국인 요양보호사에게는 여전히 장벽

---

### 플랫폼별 MVP 개발 우선순위

| 순위 | 플랫폼 | 방법 | 이유 |
|:---:|:---|:---|:---|
| **1순위** | 안드로이드 | AccessibilityService 오버레이 | 현장 요양보호사 대부분 안드로이드. 앱 전환 0번. 정책상 배포 가능. |
| **2순위** | iOS | 커스텀 키보드 익스텐션 | 외국인 요양보호사(20~30대) 아이폰 사용 가능성 대비. 앱 전환 없음. |
| **임시** | iOS | 클립보드 복사 | 키보드 익스텐션 완성 전 임시 방편. 외국인 요양보호사에게는 무난. |

---

### 이 기능이 완성됐을 때 보듬의 최종 모습

| 서류/작업 | 현재 | 보듬 완성 후 |
|:---|:---|:---|
| 공단 앱 항목 체크 | 직접 선택 | 직접 선택 (대체 불가, 법적 필수) |
| 공단 앱 서술 입력 | 직접 타이핑 | **음성 10초 → 자동 삽입 (앱 전환 0번)** |
| 전자서명 | 직접 받기 | 직접 받기 (대체 불가, 법적 필수) |
| 상태변화기록지 | 수기 작성 | **음성 → AI 자동 생성 → 기관 보관** |
| 보호자 카톡 응대 | 개인 폰 직접 작성 | **알림톡 자동 발송** |
| **타이핑 횟수** | **하루 수십 번** | **0번** |

항목 체크와 전자서명은 법적 구조상 대체가 불가능하다. 그 외 요양보호사가 손가락으로 직접 무언가를 입력해야 하는 모든 행위는 보듬이 소멸시킨다. 이것이 진정한 Zero-Typing이다.
