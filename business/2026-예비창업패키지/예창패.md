# 🏆 2026 예비창업패키지 심사 대비 — 보듬(Bodeum) 합격 전략

---

## 0. 현재 상태 진단 (냉철한 평가)

> 아이디어와 시장 정의는 우수하다. 문제는 **"증명 자료가 없다"**는 것이다.

| 평가 항목 | 현재 상태 | 문제점 |
|:---|:---:|:---|
| 문제인식 명확성 | ✅ 강점 | 팩트 기반 서술 우수 |
| 실현가능성 (기술) | ⚠️ 보통 | MVP 범위가 너무 넓음 → 1단계 집중 필요 |
| 실현가능성 (수요) | ❌ 취약 | 실제 원장님 인터뷰 증거 없음 |
| 성장전략 (BM) | ✅ 강점 | 단위 경제성 수치 구체적 |
| 팀 구성 | ⚠️ 취약 | 1인 팀 + 도메인 경험 미약 |

**핵심 교훈**: 예창패는 논리 대회가 아니라 **증거 대회**다. 심사위원은 "이렇게 될 것이다"가 아니라 "이미 확인했다"를 원한다.

---

## 1. 합격을 위한 핵심 프레이밍(Framing) 원칙

### ❌ 탈락하는 프레임
"요양원과 보호자가 소통하기 불편해서, 알림장 앱을 만들겠습니다."
→ 즉각 탈락. "케어로그, 하이케어 이미 있잖아요."

### ✅ 합격하는 프레임
**"60대 요양보호사의 스마트폰 타자 입력 불가 문제를 생성형 AI(Zero-Typing)로 해결하는 고령 친화적 B2B SaaS"**

- **딥테크 포지셔닝**: "소통 앱"이 아니라 "요양 현장 디지털 전환(DX) 인프라"
- **사회적 가치**: 요양보호사 노동 인권 개선 + 노인 인권 보호
- **린(Lean) 실행력**: 자체 AI 모델 개발 없이, 세계 최고 상용 API를 요양 도메인에 조립

## 2. 예창패 심사 항목별 대응 전략

### [항목 1: 문제인식] — 현재 강점, 유지

**핵심 팩트 3개 (반드시 외울 것)**
1. 전국 요양보호사 66만 명 중 **60세 이상 66.1%** (공단 2024년 7월 실태조사)
2. 공단 앱 외 **주 1회 서술형 관찰일지 별도 작성 의무** (건보공단 평가 기준)
3. 2028년 요양보호사 **11만 6,000여 명 부족** 전망 (건강보험연구원 2023)

**필살 한 문장**:
> "치매 어르신의 대소변을 치우고 물에 젖은 손으로 스마트폰 자판을 치는 것 — 이것이 66세 요양보호사 할머니의 현실입니다. 보듬은 그 자판 입력 행위 자체를 소멸시킵니다."

---

### [항목 2: 실현가능성] — 가장 취약, 집중 보완 필요

**약점 1: 고객 인터뷰 증거 없음** ← 가장 치명적
- 해결책: 원장님·요양보호사 최소 10건 인터뷰 실시 후 아래 형식으로 기재
  ```
  [인터뷰 사례 - 서울 은평구 방문요양센터 원장, 2026년 3월]
  "요양보호사들이 관찰일지 쓰기 싫어서 그만두는 경우가 실제로 있다.
  월 1만 원이면 당장 쓰겠다." (원장 A씨, 수급자 18명 운영)
  ```

**약점 2: MVP 범위가 너무 넓음**
- 해결책: 1단계를 "음성→알림톡 단일 파이프라인"으로 좁혀서 기재
  - 1단계 (2,000만 원): 음성 앱 + 알림톡 파이프라인 + 5개 센터 테스트
  - 2단계 (2,000만 원): GPS 넛지 + 정합성 검증 + 100개소 확장

**약점 3: "이중 작업 아닌가?" 방어**
- 공단 앱 = 법적 수가 청구 (대체 불가, 우리가 건드리지 않음)
- 보듬 = 공단 앱 **이외의** 감정노동(카톡 응대) + 수기 관찰일지 제거
- 삼중 작업 → 공단 앱 1개만 남김 = 총 작업량 감소

---

### [항목 3: 성장전략] — 강점, 더 강화 가능

**BM 설득 포인트 (숫자로 말할 것)**
- 센터 1곳 공헌이익: ~7,200원, 마진율 72%
- BEP: 전국 재가기관의 **단 3.5%(800개소)**
- "수백만 원짜리 환수 조치 방어 보험료가 월 1만 원" 프레임

**스케일업 비전 (3줄 요약)**
- Phase 1: 예창패 → 100개소 레퍼런스
- Phase 2: 소셜벤처 인증 → ESG 임팩트 펀드 Seed 투자
- Phase 3: 트로이 목마 → 공단 청구 ERP 모듈 오픈 → 이지케어 대체

---

### [항목 4: 팀 구성] — 솔직함이 오히려 신뢰

**1인 팀의 강점을 정직하게 어필**
- "외주 없이 예산 100% 인프라/마케팅에 집중"
- "AI 코딩 에이전트(Claude Code)로 개발 속도 2배"
- "의사결정 지연 없이 시장 피드백에 즉시 대응"

**약점은 먼저 인정하고 보완책 제시**
- "요양 도메인 현장 전문성은 현재 개발자 중심 — 현장 인터뷰와 자문 위원으로 보완 중"
- "2027년 스케일업 단계에서 복지/간호 전문가 영입 예정"

## 3. 서면 평가 통과 후 — 발표 평가 대비

### 발표 구성 (30분 기준)

| 시간 | 내용 |
|:---|:---|
| 0~3분 | 현장 이야기로 시작 ("물 묻은 손으로 자판 치는 66세 할머니") |
| 3~10분 | 문제: 삼중 고통 구조 + 팩트 수치 |
| 10~18분 | 해결책: Zero-Typing 시연 영상 또는 목업 |
| 18~24분 | BM + 단위 경제성 + BEP 수치 |
| 24~27분 | 로드맵 + 사회적 가치 |
| 27~30분 | 필살 클로징 멘트 |

### 클로징 3종 세트

**[사회적 가치]**
"치매 어르신을 돌보느라 불어 터진 손으로 자판을 두드리는 요양보호사의 손가락을 쉬게 해주는 것. 그것이 보듬의 존재 이유입니다."

**[실현 가능성]**
"수십억 투자가 필요 없습니다. 예창패 4천만 원으로 100개 기관을 락인하면, 이후는 데이터와 입소문이 영업합니다. 숫자로 증명된 계획입니다."

**[경쟁 우위]**
"이지케어가 700억을 벌어도 못 했던 것 — 60대 요양보호사가 앱 화면에서 손가락을 뗄 수 있게 하는 것. 보듬이 처음입니다."

## 4. 예상 Q&A 방어 대본

### Q1. "기존 경쟁사 많은데 어떻게 이깁니까?"
A: "케어로그는 타자 입력창을 예쁘게 만들었고, 하이케어는 공단 행정 연동에 집중합니다. 저희는 입력 행위 자체를 소멸시킵니다. 경쟁사가 도구를 개선할 때, 우리는 도구가 필요 없어지게 만들었습니다."

### Q2. "이중 작업 아닙니까?"
A: "공단 앱은 법적 수가 청구용으로 저희가 대체하지 않습니다. 저희가 없애는 건 공단 앱 이외에 강요받는 두 가지, 개인 카톡 보호자 응대와 주간 수기 관찰일지입니다. 총 작업량은 줄어듭니다."

### Q3. "1인 개발자가 8개월 안에 됩니까?"
A: "MVP 범위를 명확히 했습니다. 1단계에서는 '음성→알림톡 단일 파이프라인 + 5개 센터 시범 운영'만 목표입니다. OpenAI Whisper + GPT-4o-mini + 카카오 알림톡 API 조립은 현존하는 기술로 충분히 가능하고, Claude Code 등 AI 코딩 에이전트로 개발 속도를 보강합니다."

### Q4. "월 1만 원에 원장님이 결제할까요?"
A: "세 가지 중 하나만 있어도 충분합니다. 요양보호사 이직 방지 복지(이직 1명으로 날아가는 매출 >> 1만 원), 수가 환수 방어 컴플라이언스(환수 조치 수백만 원 >> 1만 원), 신규 수급자 유치 마케팅 도구. 세 가지를 동시에 제공합니다."

### Q5. "스케일업 포텐셜이 있습니까?"
A: "시작은 1만 원 B2B 구독이지만, 매일 쌓이는 어르신 상태 데이터(수면, 식사, 인지 변화)가 진짜 자산입니다. 이 데이터를 기반으로 2028년 공단 청구 모듈을 오픈하면 이지케어 고객을 흡수하는 트로이 목마 전략이 완성됩니다. 일본·독일 글로벌 시장 진출도 동일한 AI 파이프라인으로 가능합니다."
