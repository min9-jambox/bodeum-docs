# 🚀 보듬(Bodeum) 3개년 실행 로드맵 (2026~2028)

---

## 📅 Phase 1 (2026): 프론트엔드 침투 및 MVP 시장 검증

**목표**: 100개소 유료 레퍼런스 확보 + 핵심 지표 증명

**핵심 원칙**: 기능을 다 만들지 않는다. 가장 고통받는 문제 하나 — '요양보호사 음성 기록 → 보호자 알림톡' — 만 완벽하게 해결해서 빠르게 시장에 나간다.

### 예창패 1단계 (4~8월) — 반드시 완성

| 항목 | 내용 |
|:---|:---|
| 요양보호사 음성 앱 (React Native) | 단일 녹음 버튼 화면. 타이핑 없음. |
| AI 파이프라인 | 음성 → Whisper STT → GPT-4o-mini → 카카오 알림톡 |
| 원장님 웹 백오피스 | 엑셀 업로드 + 일지 확인 + ERP 호환 엑셀 다운로드 |
| 시범 운영 | 수도권 5개 센터, 요양보호사 만족도·음성 인식 정확도 수집 |

### 예창패 2단계 (9~12월) — 중간평가 통과 후

| 항목 | 내용 |
|:---|:---|
| GPS 지오펜싱 넛지 | 수급자 댁 50m 진입/이탈 시 앱 자동 알림 |
| 프리미엄 마케팅 리포트 | 원장님 직인 + 사진 갤러리 + 감성 요약 |
| 멀티모달 정합성 검증 | 음성-사진 교차 검증 엔진 (특허 출원 연계) |
| 100개소 확장 영업 | 첫 달 무료 + 1년 5,000원 프로모션 |

### 월별 세부 일정

| 월 | 핵심 과제 |
|:---|:---|
| 4~5월 | 앱/웹 기획, DB 설계, 카카오 알림톡 API 연동 테스트 |
| 6~7월 | 요양보호사 음성 앱 MVP + 원장님 웹 백오피스 완성 |
| 8월 | 수도권 5개 센터 시범 운영 + 현장 피드백 수집 |
| 9~10월 | 피드백 반영 + GPS 지오펜싱 추가 |
| 11~12월 | 정식 런칭 + 100개소 달성 목표 |

### 1차년도 핵심 지표 (KPI)

- 유료 센터 수: 100개소+
- 요양보호사 앱 DAU(일간 활성 사용자): 가입 센터의 80%+
- 보호자 알림톡 열람률: 90%+
- 월 이탈률(Churn): 5% 미만

## 📅 Phase 2 (2027): 소셜벤처 인증 + 임팩트 투자 유치

**목표**: 소셜벤처·예비사회적기업 인증 → ESG/임팩트 펀드 Seed/Pre-A 투자

**핵심 무기**: 100개 센터에서 쌓인 살아있는 데이터(요양보호사 노동 강도 감소, 보호자 알림 열람률, 센터 이탈률 감소)가 투자 설득의 핵심 근거가 된다.

| 반기 | 핵심 과제 |
|:---|:---|
| **상반기** | 기술보증기금 소셜벤처 판별 신청. 60대 요양보호사 노동 강도 감소 + 노인 인권 보호 데이터 축적으로 인증 획득. 지자체 B2G 지원 사업 연계 시작. |
| **하반기** | 확보된 Traction + 소셜벤처 인증을 무기로 VC(ESG/임팩트 펀드) IR. 목표: Seed/Pre-A 3~5억 원. 투자금으로 도메인 전문가(사회복지사·간호사 출신) + B2B 영업 전담 인력 채용. 다국어 STT 엔진 고도화(외국인 요양보호사 지원). |

## 📅 Phase 3 (2028~): 트로이 목마 완성 + Full-Suite ERP 전환

**목표**: 이지케어 등 레거시 ERP 고객 흡수 → 시장 제패

**전략**: 요양보호사(프론트엔드)와 보호자(소통 채널)를 이미 장악한 상태에서, 원장님들이 자연스럽게 기존 ERP를 버리고 보듬 ERP로 완전히 전환하도록 백엔드 기능을 순차적으로 오픈한다.

| 영역 | 핵심 과제 |
|:---|:---|
| **공단 청구 자동화** | 롱텀시스템 연동 + 원클릭 급여/본인부담금 청구 모듈 오픈 |
| **공단 평가 대비** | 평가 지표 기반 서류(급여제공기록지, 낙상위험도 등) LLM 자동 생성 |
| **복지용구 커머스 (B2B)** | 원장님 대상 복지용구 공동구매 ERP 포털 |
| **시니어 커머스 (B2C)** | 보호자 앱 내 시니어 용품 쇼핑몰 |
| **글로벌 진출** | 일본 Kaigo, 독일 DiPA 파이프라인 이식 (Phase 3, 2028~) |
