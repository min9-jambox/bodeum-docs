# 스마트장기요양 앱 분석 및 공단 연동 전략
> 작성일: 2026년 3월  
> 목적: 보듬 제품 전략 수립을 위한 현장 앱 구조 파악 및 자동화 가능성 검토

---

## 1. 스마트장기요양 앱, 직접 다운로드할 수 있나?

### 앱 존재 여부
- **안드로이드**: 구글 플레이스토어에서 "스마트장기요양" 검색 후 설치 가능
- **아이폰**: 앱스토어에 정식 등록되어 있지 않음. 공단 홈페이지(longtermcare.or.kr) 하단 QR코드 촬영 방식으로 설치

### 직접 사용 가능 여부: **불가능**
앱은 설치할 수 있지만, **2025년 리뉴얼 이후 건강보험·공동·금융 인증서 기반 로그인으로 변경**됐다.
즉, 요양보호사·수급자·보호자로 등록된 실제 자격자가 아니면 앱 내부 화면을 볼 수 없다.

> 💡 **대안**: 공단 홈페이지, 요양 관련 커뮤니티(실버케어 등), 앱스토어 리뷰 이미지에 캡처 화면이 종종 공개되어 있다. 현장 원장님·요양보호사 인터뷰 시 직접 화면을 보여달라고 요청하는 것이 가장 빠르다.

## 2. 요양보호사 출근부터 퇴근까지 전체 업무 흐름

### 공단 앱(스마트장기요양) 관련 행위

```
[수급자 댁 도착]
    ↓
① 시작 태그 전송
   - 수급자 댁에 부착된 NFC/블루투스 태그에 스마트폰 접촉
   - 서비스 시작 시간이 공단 서버에 실시간 전송 → 수가 청구 근거 생성
   - 수급자·보호자 스마트폰에 "선생님이 도착했습니다" Push 알림 발송

[서비스 제공 중: 2~4시간]
    ↓
② 돌봄 서비스 제공
   - 신체 지원: 식사 보조, 세면, 옷 갈아입기, 이동 보조, 화장실 도움 등
   - 가사 지원: 청소, 빨래, 식사 준비, 설거지 등
   - 인지 활동: 대화, 산책 동행, 레크리에이션 등

[서비스 종료 직전]
    ↓
③ 급여제공기록 입력 (공단 앱 내)
   - 2025년 리뉴얼로 기존 3종 → 14종 서식으로 확대
   - 오늘 제공한 서비스 항목을 앱에서 직접 선택·입력
   - 이것이 현장에서 가장 힘든 단계 (아이러니하게도 리뉴얼로 입력할 항목이 더 늘었다)

④ 전자서명 받기 (공단 앱 내)
   - 수급자 또는 보호자의 전자서명을 앱 화면에서 직접 받아야 함
   - 치매 어르신, 거동 불편 어르신에게 스마트폰 서명 요청이 현실적으로 매우 어렵다는 현장 불만 多

[수급자 댁 출발]
    ↓
⑤ 종료 태그 전송
   - 다시 태그에 접촉, 서비스 종료 시간 공단 서버에 전송
   - 수급자·보호자에게 "서비스가 완료됐습니다" Push 알림 발송
```

### 공단 앱 밖의 별도 업무 (보듬이 타겟하는 영역)

```
[주 1회 이상 — 기관 평가 대비]
    ↓
⑥ 수기 상태변화기록지 작성
   - 수급자의 신체·인지·정서 상태 변화를 서술형으로 작성
   - 공단 앱과 별개로 기관 내 보관 → 정기평가 시 제출
   - 현장에서 "수기로 작성하거나, 원장님이 민간 ERP에 대신 입력하거나" 방식이 혼재

[매일 — 비공식 감정노동]
    ↓
⑦ 개인 카카오톡으로 보호자 응대
   - 공단 앱의 Push 알림은 "서비스 시작·종료" 사무적 알림만 가능
   - "오늘 어머니 식사 잘 하셨고 기분 좋으셨어요 😊" 같은 감성 소통은 개인 폰으로 따로 보내야 함
   - 보호자가 회신하면 퇴근 후에도 응대해야 하는 24시간 감정노동으로 이어짐
```

### 현장 반응 (앱스토어 리뷰 요약)
> "대부분 요양보호사 연령대가 60·70대인데 다들 너무 힘들어 하신다, 알려드리고 싶어도 제가 설치한 앱조차 열리지 않는다, 오셨다가 그냥 가시는 분이 대부분"

> "솔직히 사용하라고 만든 앱인지 의문이 드는 수준입니다. 사용하는 선생님들의 연세가 50~70대인데 그걸 고려하고 만든건지 궁금합니다."

## 3. 핵심 질문: 보듬에서 음성 입력 → 공단 앱 14가지 자동 입력이 가능한가?

### 결론부터: **현시점 직접 자동화는 불가능. 단, 우회 경로가 있다.**

### 왜 불가능한가?

공단이 외부 앱을 위한 **급여제공기록 '쓰기(Write)' API를 공개하지 않았기 때문**이다.

공공데이터포털(data.go.kr)에 국민건강보험공단의 Open API가 일부 존재하지만, 이는 전부 **기관 정보 조회, 등급 판정 현황 등 '읽기(Read)' 전용**이다. 스마트장기요양 앱 내 급여제공기록을 외부에서 대신 써주는(Write) API는 공식적으로 존재하지 않는다.

즉, 지금 공단 앱과 직접 연동해서 기록을 자동 입력하는 것은:
- 공식 API 경로: **없음**
- 스크래핑/자동화 봇 경로: **기술적으로는 가능하나 법적 리스크 + 유지보수 지옥**

### 하이케어가 쓰는 방법 (스크래핑)

하이케어 등 일부 민간 ERP가 '공단 롱텀 자동 연동'을 제공하는 방식은 공식 API가 아닌 **RPA(로봇 프로세스 자동화) 또는 웹 스크래핑** 방식으로 추정된다. 이는 공인인증서를 위임받아 공단 시스템에 대신 접속하는 구조다.

문제점:
1. **법적 리스크**: 공단이 명시적으로 허용하지 않은 자동 접속은 전산망 이용에 관한 법령 위반 소지
2. **유지보수 지옥**: 공단이 앱이나 웹 UI를 조금만 바꿔도 전체 스크래핑 로직이 깨짐 (실제로 2025년 리뉴얼 시 하이케어 연동 오류 현장 불만 다수 발생)
3. **인증서 보안 문제**: 요양보호사 개인 인증서를 민간 앱에 위탁하는 구조

## 4. 보듬의 현실적 전략 — 3단계 접근

### 1단계 (MVP, 2026년): 공단 앱과 분리해서 가치 증명

**공단 앱 14종 입력은 건드리지 않는다.** 보듬은 공단 앱 이외에 존재하는 두 가지 고통만 없앤다.
- ⑥ 수기 상태변화기록지 → 음성으로 AI 자동 생성
- ⑦ 개인 카톡 보호자 응대 → 알림톡 자동 발송으로 대체

이것만으로도 요양보호사의 퇴근 후 감정노동과 주간 수기 작성이 사라진다. 충분히 강력한 가치다.

---

### 2단계 (2027~): 공단과 파트너십 협력 추진

공단이 직접 API를 열어줄 가능성이 생기는 경로가 있다.

**경로 A — 소셜벤처·정부 R&D 트랙 활용**
보건복지부·공단은 "스마트 돌봄 플랫폼 생태계 조성"을 정책 목표로 삼고 있다. 소셜벤처 인증 후 공단의 디지털 혁신 파트너로 등록되면, 공단 내부 API(제한적 접근)를 협약 기반으로 활용할 수 있는 통로가 열릴 수 있다.

**경로 B — 공단 앱의 '급여제공계획 확인' API 활용 (읽기 전용)**
현재도 공개된 읽기 전용 API를 통해 수급자의 서비스 계획 정보를 가져올 수 있다. 이를 보듬에서 미리 받아와, 요양보호사가 음성으로 말하면 **해당 계획과 비교해 맞는 항목을 자동 매칭**하는 구조를 만들 수 있다.

---

### 3단계 (2028~): 공단 앱 14종 입력 자동화 — 트로이 목마 완성

이 단계에서 보듬이 노리는 것은 단순한 자동화가 아니다. **공단이 자체적으로 음성 기반 입력 인터페이스를 구축하거나, 보듬 같은 솔루션과 공식 파트너십을 맺는 것이 정책적으로 더 합리적인 시점이 온다.**

왜냐하면:
- 공단 스스로도 60~70대 요양보호사의 앱 사용 불만을 인식하고 있다
- 앱스토어 리뷰 불만이 쌓이고, 현장 이탈이 증가하면 공단도 UX 개선 압박을 받는다
- 이때 "음성 입력 기술을 가진 파트너"로 보듬이 포지셔닝돼 있으면, 공식 API 파트너 협약이 가능해진다

## 5. 이 발견이 보듬 사업계획서에 주는 시사점

### 기존 메시지의 문제점 (솔직하게)
"보듬이 입력 행위를 소멸시킨다"고 말하지만, 공단 앱 14종 입력은 여전히 요양보호사가 직접 해야 한다. 심사위원이 이 점을 지적하면 "공단 앱 영역은 건드리지 않는다"는 대답이 되고, 그러면 "그럼 입력이 완전히 없어지는 건 아니잖아요?"라는 반박이 나온다.

### 수정된 메시지 (정확한 포지셔닝)

> **"공단 앱이 담당하는 수가 청구 영역은 법적으로 대체 불가합니다. 보듬은 그 영역을 건드리지 않습니다. 대신 공단 앱이 전혀 해결하지 못하는 두 가지 — 주 1회 이상 수기 상태변화기록지와 매일 개인 카톡 보호자 응대 — 를 소멸시킵니다. 이 두 가지가 요양보호사가 '퇴근 후에도 일해야 하는' 감정노동의 본체입니다."**

즉, "모든 입력을 없앤다"가 아니라 **"공단 앱이 커버하지 못하는 비공식 행정 고통을 없앤다"** 로 포지셔닝을 명확히 해야 한다. 이것이 더 정직하고, 심사위원의 날카로운 지적을 피하는 더 강한 논리다.

### 장기 비전으로 추가할 문구

> "2단계(2027년~)에는 건보공단과의 파트너십 협력을 통해, 보듬에서 AI가 생성한 급여제공기록 데이터를 공단 앱과 연동하는 구조를 추진합니다. 공단 스스로도 현장 요양보호사의 앱 사용 불만을 인식하고 있으며, 음성 기반 입력 기술과의 협력은 공단의 정책 방향과도 일치합니다."

## 6. 요약 정리

| 항목 | 현황 |
|:---|:---|
| 앱 다운로드 가능 여부 | 안드로이드 가능, 아이폰 QR 설치 |
| 실제 사용 가능 여부 | 인증서 로그인 필요 → 일반인 접근 불가 |
| 공단 앱 주요 기능 | 시작·종료 태그, 14종 급여제공기록 입력, 전자서명, 수급자 일정 확인 |
| 공단 앱이 담당하지 않는 것 | 서술형 상태변화기록지(주 1회), 보호자 감성 소통 |
| 외부 앱 → 공단 앱 자동 입력 | **공식 API 없음**. 스크래핑은 법적 리스크 + 유지보수 리스크 |
| 보듬 1단계 타겟 | 공단 앱 밖의 수기 기록 + 카톡 감정노동 소멸 |
| 보듬 2단계 목표 | 공단 파트너십 → 급여제공기록 연동 API 확보 |
| 보듬 3단계 비전 | 공단 앱 대체 가능한 올인원 인터페이스 |
