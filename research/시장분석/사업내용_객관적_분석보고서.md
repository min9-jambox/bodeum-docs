# SilverBridge / 보듬 — 사업 타당성 팩트 기반 객관적 분석 보고서

> **작성 목적**: 예창패 제출 전 사업기획서의 논리적 취약점을 사전에 파악하고, 심사위원 관점에서 보완해야 할 부분을 냉철하게 정리한 내부 검토 자료.

---

## 1. 시장 및 경쟁사 현황 — 팩트 기반

### ① 거대 자본 플레이어들이 이미 존재한다

**이로움돌봄(티에이치케이컴퍼니)**: 누적 투자 210억 원 이상, 2023년 연매출 527억 원, B2B 복지용구 유통 국내 1위. **케어링**: 요양보호사 4만 2,000명, 누적 투자 750억 원(시리즈 B). **이지케어텍**: 2024년 매출 726억 원, 재가기관 ERP 1위.

**분석**: 이들과 정면 충돌하는 '통합 플랫폼' 또는 '풀스위트 ERP' 포지셔닝은 1인 개발자에게는 즉각 필패다. 단, 이들이 모두 **해결하지 않은 영역 — '60대 요양보호사의 타자 입력 행위 자체'** — 은 여전히 공백으로 남아 있다.

### ② 직접 경쟁사 케어로그의 현황과 한계

케어로그는 구글·중소벤처기업부 창구 프로그램 7기 선정(2025년 4월), 중기부 디딤돌 R&D 과제 완료(2024년), 음성·사진 기반 일지 특허 보유. 방문요양 특화 소통·일지 앱으로 보듬과 표면적 기능이 가장 겹치는 경쟁사다.

**케어로그의 구조적 약점 (보듬이 이길 수 있는 이유):**

1. **채팅 기능의 역설**: 보호자-요양보호사 간 양방향 채팅 기능은 기존 카톡 감정노동을 앱 안으로 이식한 것에 불과하다. 요양보호사에게는 새로운 민원 채널이 하나 더 생긴 셈.
2. **보호자 앱 설치 강제**: 앱스토어 검색 → 다운로드 → 회원가입 → 센터 승인의 3단계가 필수. 보호자 전환율 허들이 높다.
3. **타자 입력 강제**: 아무리 UI가 개선돼도 결국 스마트폰 자판을 쳐야 한다.

**보듬의 핵심 차별화**: 입력 행위 자체를 소멸 + 단방향 알림톡(무설치) + 기존 ERP Add-on 포지셔닝

### ③ 기존 레거시 ERP의 수비력 — 우리가 싸울 필요 없다

이지케어, 케어포 등은 원장님의 청구·행정(백엔드)에 집중되어 있다. 이들과 전면전을 치르는 것은 1인 개발자에게 불가능하다. 그러나 이들이 방치한 **'요양보호사 프론트엔드(음성 기록 + 가족 소통)'** 레이어는 비어 있다.

**결론**: 보듬은 기존 ERP와 싸우는 게 아니라, 그 위에 얹히는 Add-on이다.

## 2. 예창패 합격 가능성 냉정 평가

**현재 상태**: 서류 통과 가능성 있으나 최종 선정은 조건부. 핵심 문제는 **"논리는 있는데 증거가 없다"**.

### 탈락하는 프레임 (절대 피할 것)
> "요양원과 보호자가 소통하기 불편해서 알림장 앱을 만들겠습니다."

→ 심사위원 반응: "케어로그, 하이케어, 패밀리노트 다 있는데 1인 개발자가 어떻게 이깁니까?"

### 합격하는 프레임 (이것으로 포지셔닝)
> **"60대 요양보호사의 스마트폰 타자 입력 불가 문제를 생성형 AI(Zero-Typing)로 해결하는 고령 친화적 B2B SaaS"**

- 딥테크/AI 기술 포지셔닝: "소통 앱"이 아니라 "요양 현장 DX 인프라"
- ESG + 사회적 가치: 60대 노동자 인권 보호
- 린(Lean) 실행력: 자체 모델 개발 없이 상용 API 조립으로 원가 최소화

### 합격률을 높이는 핵심 논리 3가지

**1. 경쟁사들이 포기한 '입력단' 문제 해결 (Zero-Typing)**

패밀리노트, 케어로그 등 모든 경쟁사는 UI는 개선했지만 타자 입력 자체는 그대로 강제한다. 보듬은 사투리로 10초만 말하면 AI가 ① 관찰일지 서식 ② 보호자 알림톡을 동시에 완성한다. 이것은 어떤 경쟁사도 하지 않은 접근이다.

**2. 파괴적 요금제 (현실적 실행 능력)**

첫 달 무료 + 월 1만 원 + 알림톡 실비 종량제. 센터 1곳 공헌이익 7,200원, 마진율 72%. 전국 재가기관의 3.5%(800개소)만 확보하면 월 순수익 500만 원. 예창패 자금 활용 방법(알림톡 무상 프로모션 1,000만 원)이 구체적이다.

**3. 보호자 앱 설치 포기 선언 (카카오 피기백)**

보호자 전용 앱을 만들지 않는다. 카카오 알림톡으로만 발송. 설치 마찰 0, 열람률 100% 접근. 막대한 CAC(고객획득비용) 없이 운영 가능.

## 3. 치명적 약점과 해결 방법

### 약점 1: 고객 인터뷰 증거 없음 (가장 치명적)

현재 기획서 전체에 실제 원장님·요양보호사 인터뷰가 없다. 예창패는 논리 대회가 아니라 **증거 대회**다.

**해결**: 지금 당장 원장님 10명에게 연락해서 15분 인터뷰. 날짜, 직책, 핵심 발언을 기록해서 사업계획서에 삽입.

### 약점 2: MVP 범위 과다

음성 앱 + GPS + 정합성 검증 + 다국어 + W4C 매핑을 1인이 4개월 안에 만든다는 것은 심사위원이 믿지 않는다.

**해결**: 1단계 MVP를 "음성→알림톡 단일 파이프라인 + 5개 센터 시범 운영"으로 좁혀라. 나머지는 2단계로 분리.

### 약점 3: 팀 구성 취약

1인 팀 + 복지 도메인 경험 부족.

**해결**: 솔직하게 약점을 인정하고, "1인의 강점(외주 없이 예산 집중, 의사결정 속도)"과 "보완 방안(도메인 자문위원, 2027년 전문가 영입 계획)"을 함께 서술.
