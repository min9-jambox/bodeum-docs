# 보듬(Bodeum) 필수 반영 기능 목록

> 패밀리케어 ERP 111장 화면 분석 기반
> 작성일: 2026-03-20
> 목적: 보듬 MVP 및 로드맵에 반드시 포함해야 할 기능 도출

---

## 판단 기준

패밀리케어 ERP 분석 결과, 재가 장기요양기관이 실제로 사용하는 기능의 범위가 명확해졌다.
보듬은 "AI 기록 자동화"로 시작하지만, 결국 기관이 일상적으로 쓰는 ERP 기능과 **자연스럽게 연결**되어야 장기 고착(retention)이 발생한다.

아래 기능들을 3단계로 분류한다:
- **Phase 1 (MVP)**: 보듬의 핵심 차별점. 패밀리케어가 못하는 것 + 기관이 가장 고통받는 것
- **Phase 2 (고도화)**: 유료 전환 후 기관이 기대하는 확장 기능
- **Phase 3 (올인원 전환)**: 패밀리케어 대체를 위한 ERP 기능

---

## Phase 1 — MVP 필수 기능 (협약 후 6개월)

보듬의 존재 이유. 패밀리케어 대비 압도적 차별점이 되는 기능.

### 1-1. AI 음성 → 상태변화 기록지 자동 작성 ⭐⭐⭐

| 항목 | 내용 |
|------|------|
| 패밀리케어 현황 | 수기 텍스트 입력. 날짜·수급자 선택 후 직접 타이핑 |
| 보듬 차별점 | 요양보호사가 음성으로 말하면 → Whisper STT → GPT-4o-mini → 서술형 기록지 자동 생성 |
| 왜 필수인가 | 이것이 보듬의 핵심 가치제안(VP). 주 1회 30분+ 걸리는 수기 작성을 5분 이내로 단축 |

**구현 시 참고할 패밀리케어 필드:**
- 기록일자
- 수급자명
- 작성자(복지사/요양보호사)
- 서술형 본문 (건강상태 변화, 특이사항, 조치사항)

### 1-2. 카카오 알림톡 — 보호자 자동 알림 ⭐⭐⭐

| 항목 | 내용 |
|------|------|
| 패밀리케어 현황 | 일정 알림톡 발송 (월별 일정표 일괄 발송), 급여명세서 발송, 영수증 발송 |
| 보듬 차별점 | 케어 기록 기반 보호자 맞춤 알림. "오늘 어머님 컨디션 좋으십니다" 등 AI 요약 발송 |
| 왜 필수인가 | 패밀리케어의 알림톡은 행정 알림(일정, 비용)만 가능. 보듬은 "케어 내용" 알림으로 감정적 가치 제공 |

**패밀리케어 알림톡 발송 화면에서 확인된 필드:**
- 수급자명, 급여종류, 수급자 전화번호, 발송 유무
- 보듬은 여기에 "케어 요약 메시지"를 추가

### 1-3. 수급자 기본 정보 관리 ⭐⭐

| 항목 | 내용 |
|------|------|
| 패밀리케어 현황 | 이름, 등급, 인정번호, 인정유효기간, 사진, 주소, 보호자 연락처, 급여 이용 현황 |
| 보듬 MVP 범위 | 최소한의 수급자 정보 (이름, 등급, 보호자 연락처) — 기록지와 알림톡 발송에 필요한 정도 |
| 왜 필수인가 | 기록지 작성과 알림톡 발송의 기본 데이터. 이것 없이는 서비스가 동작하지 않음 |

### 1-4. 요양보호사 기본 정보 관리 ⭐⭐

| 항목 | 내용 |
|------|------|
| 패밀리케어 현황 | 직원 정보관리 (이름, 생년월일, 담당직무, 4대보험, 배상책임보험, 상태이력 등) |
| 보듬 MVP 범위 | 최소한의 보호사 정보 (이름, 연락처, 담당 수급자 매핑) |
| 왜 필수인가 | 누가 어떤 수급자를 담당하는지 매핑이 있어야 기록지 자동 작성이 가능 |

### 1-5. 상담일지 연계 ⭐

| 항목 | 내용 |
|------|------|
| 패밀리케어 현황 | 상담일지 목록 + 급여제공반영 연계 기능 |
| 보듬 MVP 범위 | 음성 기록에서 상담 내용이 감지되면 자동으로 상담일지 항목도 생성 |
| 왜 필수인가 | 기관 평가 시 상담일지도 중요 서류. 기록지와 함께 자동화하면 가치 2배 |

## Phase 2 — 유료 전환 후 확장 기능 (6~12개월)

기관이 보듬을 "단순 기록 도구"가 아니라 "운영 도구"로 인식하게 만드는 기능.

### 2-1. 일정관리 — 수급자·보호사 일정 ⭐⭐

| 항목 | 내용 |
|------|------|
| 패밀리케어 현황 | 캘린더 기반 일정관리, 일/주/월 뷰, 보호사·복지사·수급자별 일정 |
| 보듬 방향 | 간소화된 일정 캘린더. 기록 데이터와 연동하여 "오늘 방문 대상" 자동 표시 |
| 구현 우선순위 | 높음. 기관장이 매일 가장 많이 보는 화면 |

**패밀리케어에서 확인된 일정관리 핵심 기능:**
- 수급자 일정관리 (급여 이용 계획 매트릭스)
- 요양보호사 근무 일정 (캘린더)
- 보호사 일정 한눈에 보기 (일/주/월 전환)
- 복지사 담당수급자 지정
- 라운딩 일정 (복지사 방문 캘린더)

### 2-2. 기초평가 관리 ⭐⭐

| 항목 | 내용 |
|------|------|
| 패밀리케어 현황 | 수급자별 기초평가 시기 관리, 미실시대상 알람 |
| 보듬 방향 | 기초평가 일정 자동 알림 + AI가 과거 기록 기반 평가 초안 제안 |
| 왜 필요한가 | 기관 평가 핵심 항목. 놓치면 감점 |

### 2-3. 근무일지 및 출근부 관리 ⭐

| 항목 | 내용 |
|------|------|
| 패밀리케어 현황 | 캘린더 기반 근무일정, 일괄등록, 출근부 전체 출력 |
| 보듬 방향 | 음성 기록의 시간 데이터를 활용하여 출퇴근 자동 기록 연계 가능 |

### 2-4. 사례회의록 / 직원회의록 자동 작성 ⭐

| 항목 | 내용 |
|------|------|
| 패밀리케어 현황 | 수기 작성. 회의 주제, 참석자, 내용 등을 직접 입력 |
| 보듬 방향 | 회의 음성 녹음 → AI 회의록 자동 작성. 기관 평가용 서류 자동화 확장 |

### 2-5. 수급자 안전관리 자료제공 이력 ⭐

| 항목 | 내용 |
|------|------|
| 패밀리케어 현황 | 안내자료 제공 이력 관리, 출력 |
| 보듬 방향 | 기관 평가 대비 자동 이력 관리 |

## Phase 3 — 올인원 ERP 전환 (12개월+)

패밀리케어를 완전히 대체할 수 있는 수준의 기능. 사업계획서 Phase 3 "올인원 기관 운영 SaaS"와 일치.

### 3-1. 급여제공 — 공단 엑셀 연동 ⭐⭐⭐

| 항목 | 내용 |
|------|------|
| 패밀리케어 현황 | 청구내역상세, RFID 실적 연동, 일정계획 공단 연동 — 모두 엑셀 다운로드/업로드 방식 |
| 보듬 방향 | 공단 엑셀 파일 임포트/익스포트 연동. 기록 데이터 → 공단 엑셀 양식으로 자동 변환. 공단은 운영 데이터 API를 제공하지 않으므로 엑셀 교환 방식이 유일한 연동 수단 |
| 난이도 | 중간. 공단 엑셀 양식 파싱·생성 로직 개발 필요. API 인증은 불필요(엑셀 파일 기반) |

### 3-2. 본인부담금 관리 ⭐⭐

| 항목 | 내용 |
|------|------|
| 패밀리케어 현황 | 청구관리, 입금관리, 영수증 발송, 납부확인서, 환불처리 |
| 보듬 방향 | 청구서 자동 생성 + 알림톡 발송 통합 |

### 3-3. 직원 월 급여 관리 ⭐

| 항목 | 내용 |
|------|------|
| 패밀리케어 현황 | 급여 기초설정(시급/월급), 월 급여대장, 명세서 발송, 연간대장, 퇴직적립금 |
| 보듬 방향 | 급여 자동 산출 + 명세서 알림톡 발송 |
| 참고 | 시급제/월급제 구분, 4대보험 자동 계산, 수당/공제 항목 커스터마이징 필요 |

### 3-4. 운영관리 ⭐

| 항목 | 내용 |
|------|------|
| 패밀리케어 현황 | 기관정보 설정, 차량/욕조/비품관리, 배상책임보험, 메뉴 권한 |
| 보듬 방향 | 기관 기본 설정 + 권한 관리 정도만 우선 구현 |

### 3-5. 부가서비스 — 서식자료실 / 업무 가이드 ⭐

| 항목 | 내용 |
|------|------|
| 패밀리케어 현황 | 서식 다운로드, 일일/주간/월간 업무 체크리스트 |
| 보듬 방향 | AI 기반 업무 가이드 (오늘 해야 할 일 자동 추천) |

## 핵심 인사이트 요약

### 보듬이 패밀리케어를 이기는 포인트

| 영역 | 패밀리케어 | 보듬 차별점 |
|------|-----------|-----------|
| 상태변화 기록지 | 수기 타이핑 | **AI 음성 → 자동 작성** |
| 보호자 소통 | 일정/비용 알림톡 (일방향) | **케어 내용 AI 요약 알림 (감정적 가치)** |
| 상담일지 | 수기 작성 | **음성 기록 연계 자동 생성** |
| 사례회의록 | 수기 작성 | **회의 녹음 → 자동 회의록** |
| 데이터 분석 | 단순 그래프 | **AI 인사이트 (건강 추이, 리스크 알림)** |
| UX | 데스크톱 웹 (테이블 중심) | **모바일 퍼스트 (음성 중심)** |

### 보듬이 패밀리케어에서 배워야 할 포인트

| 영역 | 패밀리케어의 강점 | 보듬 반영 방향 |
|------|----------------|-------------|
| 공단 엑셀 연동 | 일정계획, 청구내역 엑셀 다운로드/업로드 양방향 연동 (API 미제공) | Phase 3에서 반드시 구현 |
| 인쇄/출력 | 거의 모든 화면에 출력 기능 | 기관 평가 대비 PDF 출력 필수 |
| 업무 가이드 | 일일/주간/월간 체크리스트 | AI가 자동으로 "오늘의 할 일" 추천 |
| 알림톡 범위 | 일정, 급여명세서, 영수증 등 다양 | Phase 2부터 알림톡 용도 확장 |
| 급여 관리 | 시급/월급 자동 계산, 4대보험 | Phase 3 올인원 전환 시 필수 |
| 서식자료실 | 기관 운영 필수 양식 제공 | 부가가치 서비스로 제공 가능 |

## MVP 체크리스트 (Phase 1 최소 구현 범위)

- [ ] 수급자 등록/관리 (이름, 등급, 보호자 연락처)
- [ ] 요양보호사 등록/관리 (이름, 연락처, 담당 수급자)
- [ ] 음성 녹음 → STT → AI 기록지 자동 생성
- [ ] 상태변화 기록지 CRUD (생성/조회/수정/삭제)
- [ ] 상담일지 자동 생성 연계
- [ ] 기록지 PDF 출력 (기관 평가 대비)
- [ ] 카카오 알림톡 보호자 발송 (케어 요약)
- [ ] 기록 이력 조회 (날짜별, 수급자별)
- [ ] 기관 기본 설정 (기관명, 기관번호)
- [ ] 모바일 앱 (React Native) — 보호사 음성 입력용
- [ ] 웹 관리자 페이지 (React) — 기관장/복지사 관리용
